# Example: Proposing Approaches

After understanding requirements, propose 2-3 implementation approaches conversationally.

## Structure

```
Based on our discussion, I see 3 approaches:

**Approach A (Recommended):** [Description]
- Pros: ...
- Cons: ...

**Approach B:** [Description]
- Pros: ...
- Cons: ...

**Approach C:** [Description]
- Pros: ...
- Cons: ...

I recommend Approach A because [reasons]. What do you think?
```

## Tips

- **Lead with your recommended option** and explain why
- **Include trade-offs** for each approach
- Let user make the final decision
