# 精简 Skill 任务文档

## Problem

`quick-tasks/SKILL.md`（170行）和 `breakdown-tasks/SKILL.md`（479行）中包含大量关于 CLI 自动生成任务的详细描述：

- Test task responsibility chain（T-quick-1~5、T-test-1~5 的行为描述）
- Fix-task 命令模板和去重规则
- Profile-suffix 规则和多 profile 展开逻辑
- 13 种 task type 定义表
- Gate/summary 自动生成规则

这些信息对**执行 skill 生成 business tasks 的 agent** 是噪音——agent 只需运行 `forge task index` 就够了。详细描述是给开发者和 task-executor agent 参考的，不应占据 skill 执行路径。

**现状量化**：

| 文件 | 总行数 | CLI 任务描述行数 | 占比 |
|------|--------|-----------------|------|
| quick-tasks/SKILL.md | ~170 | ~70 (Step 4 + Output Checklist) | ~41% |
| breakdown-tasks/SKILL.md | ~479 | ~180 (Steps 4b/4c/4d + type table) | ~38% |

## Proposed Solution

1. **精简 SKILL.md**：将 CLI 自动生成任务的详细描述替换为简短指令（"run `forge task index`，它自动处理"），保留 agent 执行 skill 时必须的信息
2. **补充 README.md**：将提取的内容作为开发者参考补充到 README.md 新增的"Task Types 参考"章节

### 具体变更

**quick-tasks/SKILL.md**：
- Step 4 (Test Tasks)：移除 responsibility chain、fix-task 命令模板、profile-suffix 说明，替换为 `Test tasks are auto-generated by forge task index. Do NOT create test task .md files manually.`
- Output Checklist：移除 test-task 相关 checklist items
- 保留：Step 0 Profile Resolution（agent 需要解析 profile）、fix-task 的 `forge task add` 一行命令

**breakdown-tasks/SKILL.md**：
- Steps 4b/4c (Phase Summary & Gate Tasks)：合并为一句话 `Stage-gate files are auto-generated by forge task index for phases with >=2 business tasks.`
- Step 4d (Standard Test Tasks)：移除 responsibility chain、per-profile 展开规则、fix-task 命令模板，替换为简短指令
- 13 种 task type 表：移除（agent 生成 business tasks 时不需知道这些类型）
- 保留：Step 0 Profile Resolution、fix-task 的 `forge task add` 一行命令

**README.md**：
- 在"文档索引"前新增"Task Types & Pipeline 参考"章节，包含：
  - 13 种 task type 表（type + 谁生成 + 用途）
  - Quick pipeline responsibility chain（T-quick-1~5）
  - Full pipeline responsibility chain（T-test-1~5）
  - Fix-task 命令模板和去重规则
  - Profile-suffix 规则
  - Gate/summary 自动生成规则

## Alternatives

| 方案 | 优点 | 缺点 |
|------|------|------|
| **提取到 README.md**（推荐） | 开发者自然发现；不新增文件 | README 略变长 |
| 提取到独立参考文档 | SKILL.md 最精简 | 新增文件，维护分散 |
| 就地压缩 | 变更最小 | 仍有冗余信息占用 context window |
| 不做 | 零风险 | agent context window 浪费 ~38% |

## Scope

### In Scope
- `plugins/forge/skills/quick-tasks/SKILL.md` 精简
- `plugins/forge/skills/breakdown-tasks/SKILL.md` 精简
- `README.md` 新增 Task Types 参考章节

### Out of Scope
- 任务模板文件（task.md、task-doc.md）不变——frontmatter 字段有 CLI 消费者，body sections 有下游依赖
- task-executor agent prompt（如需引用参考内容，是独立优化）
- 其他 SKILL 文件

## Risks

| 风险 | 缓解 |
|------|------|
| 精简过度导致 agent 不理解 `forge task index` 做了什么 | 保留关键指令："auto-generates test tasks, gates, summaries"，不删到零 |
| README.md 变长影响可读性 | 放在文档索引前，用折叠表格保持紧凑 |
| 提取内容与 CLI 实际行为不同步 | README 标注"由 forge task index 自动生成，以 CLI 行为准" |

## Success Criteria

- [ ] quick-tasks/SKILL.md 行数减少 >= 30%（170 → ~120）
- [ ] breakdown-tasks/SKILL.md 行数减少 >= 25%（479 → ~360）
- [ ] README.md 新增完整的 Task Types 参考章节
- [ ] 精简后的 SKILL.md 仍包含 agent 执行 skill 所需的全部指令
- [ ] 无功能性丢失——精简内容全部可在 README.md 中找到
