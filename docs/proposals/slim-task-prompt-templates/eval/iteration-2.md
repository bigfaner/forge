## Phase 3: Blindspot Hunt

1. **[blindspot] SC-FM-4's grep verification is insufficiently scoped**: The success criterion states `grep -L "PhaseSummary" forge-cli/pkg/prompt/templates/*.md` should output templates where "frontmatter 部分不包含 PhaseSummary" (line 525). However, `grep -L` matches files that do NOT contain the pattern anywhere — it will falsely flag templates where PhaseSummary has been properly migrated to the body section (which contains `## PhaseSummary`). The verification method cannot distinguish between "PhaseSummary absent from frontmatter" and "PhaseSummary absent from the entire file". — Quote: line 525: "grep -L 'PhaseSummary' forge-cli/pkg/prompt/templates/*.md 输出 frontmatter 部分不包含 PhaseSummary". — What must improve: Use a scoped frontmatter extraction + grep pipeline (e.g., extract content between first and second `---`, then grep within that range), or restructure the verification as two independent checks: (a) no PhaseSummary in frontmatter section, (b) `## PhaseSummary` section exists in body.

2. **[blindspot] The "30% debugging time reduction" estimate is unattributed**: The frontmatter benefit quantification states "预估减少调试时间约 30%——当前模板变量拼写错误需在运行时通过输出缺失字段排查，分组校验在启动时即报错" (line 45). This is the only quantitative benefit claim for frontmatter restructuring, but the 30% figure has no attribution — no debugging logs, no incident reports, no developer time tracking. If this estimate is an assumption rather than data, it should be labeled as such and the proposal should define a validation method (e.g., "track time spent on template variable debugging in the 2 weeks before and after deployment"). Without validation, the frontmatter work stream's ROI rests on an unverified assumption. — Quote: line 45: "预估减少调试时间约 30%——当前模板变量拼写错误需在运行时通过输出缺失字段排查". — What must improve: Either attribute the 30% to historical data or label it as an assumption and add a post-deployment validation metric to the SC set.

3. **[blindspot] No risk covering cascading rework between work streams**: The execution ordering constraint (lines 57-59) defines the sequence (content trimming first, frontmatter second) and baseline invalidation rules. But it does not address the risk that frontmatter restructuring (stream b) may discover issues requiring content re-trimming. For example, if PhaseSummary section migration (SC-FM-4) reveals that the PhaseSummary content interacts with the surrounding trimmed text in a way that changes agent behavior, the content trimming baseline would need to be re-established. The proposal treats the two streams as sequential but independent, when they modify the same files and could have emergent interactions. — Quote: lines 57-59: "执行顺序为：(a) 内容精简先执行，完成后以精简后的模板建立 SC-Pre 基线；(b) Frontmatter 重构随后执行...两线各自独立验证，避免基线交叉污染." — What must improve: Add a risk entry for "Stream (b) frontmatter changes trigger re-baselining of stream (a) content trimming SC-Pre" with mitigation (e.g., "if stream (b) modifies body content beyond PhaseSummary section, re-run SC1 snapshot comparison against the post-(a) baseline").

4. **[blindspot] The Out of Scope "不改动模板占位符" item is technically violated by PhaseSummary conditional block**: The Out of Scope section states "不改动模板占位符（`{{.X}}` 语法不变）" (line 443). However, the PhaseSummary section migration adds a new `{{if .PhaseSummary}}## PhaseSummary\n{{.PhaseSummary}}{{end}}` conditional block (lines 204-206) to all 21 prompt templates. While `{{.X}}` syntax is preserved, this is a new template construct — a conditional block that did not previously exist in the body (the PhaseSummary was presumably rendered through a different mechanism). The Out of Scope item is over-broadly stated; it likely means "no changes to existing placeholder patterns" rather than "no new template constructs", but the current wording creates a verifiable inconsistency. — Quote: line 443: "不改动模板占位符（`{{.X}}` 语法不变）"; lines 204-206: `{{if .PhaseSummary}}## PhaseSummary\n{{.PhaseSummary}}{{end}}`. — What must improve: Refine the Out of Scope item to "不改动现有模板占位符和条件块的语法结构；新增 PhaseSummary 条件块为 In Scope 的例外" or equivalently precise language.
