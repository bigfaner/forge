# 2.4: Submit command refactor

## Description

Refactor `internal/cmd/submit.go` to use state machine validation + `WithLock` callback. Remove `--force` flag. Submit already has lock+atomic writes — this task replaces manual lock management with `WithLock` and adds state machine validation.

## Reference Files

- design/tech-design.md — Interface 1 caller pattern, Error Handling § submit.go row
- design/state-transition-diagram.md — RoleSubmit rules
- `forge-cli/internal/cmd/submit.go` — current implementation (manual LockFile/UnlockFile, --force flag)

## Acceptance Criteria

- [ ] `--force` flag completely removed
- [ ] Manual `LockFile`/`UnlockFile` replaced with `WithLock(indexPath, fn)`
- [ ] Submit calls `ValidateTransition(current, "completed", RoleSubmit)` — rejects terminal states
- [ ] Auto-downgrade calls `ValidateTransition(current, "blocked", RoleSubmit)` — sets BlockedReason
- [ ] Lock conflict → `ErrLockConflict` (exit 1), invalid transition → `ErrInvalidTransition` (exit 2)
- [ ] Characterization test `TestSubmit_CurrentBehavior_AllowsCompletedResubmit` updated to expect error
- [ ] `go build ./...` + `go test ./internal/cmd/...` pass

## Hard Rules

- `--force` must be completely removed, not hidden/deprecated
- BlockedReason format: `"auto-downgrade: testsFailed=N"`

## Implementation Notes

Submit already uses `SaveIndexAtomic` — only the lock management pattern changes (manual → WithLock callback). The state machine validation is the new behavior. Quality gate validation stays but no longer has a `--force` escape hatch.
