# 2.4: Submit command refactor

## Description

Refactor `cmd/submit.go` to use `ValidateTransition` + `WithLock` + AIError. Remove `--force` flag. Submit on completed/rejected/skipped tasks returns `ErrInvalidTransition` (exit 2). Auto-downgrade sets `BlockedReason`.

## Reference Files

- design/tech-design.md — Interface 1 caller pattern (submit.go example), Error Handling § per-file migration (submit.go row)
- design/state-transition-diagram.md — RoleSubmit 规则

## Acceptance Criteria

- [ ] `--force` flag removed from submit command
- [ ] Submit calls `ValidateTransition(current, "completed", RoleSubmit)` — rejects terminal states
- [ ] Submit calls `ValidateTransition(current, "blocked", RoleSubmit)` on auto-downgrade — sets BlockedReason
- [ ] Index writes use `WithLock(indexPath, fn)` pattern
- [ ] Lock conflict → `ErrLockConflict` (exit 1), invalid transition → `ErrInvalidTransition` (exit 2)
- [ ] Characterization test `TestSubmit_CurrentBehavior_AllowsCompletedResubmit` updated to expect error
- [ ] `go build ./...` + `go test ./cmd/...` pass

## User Stories

- Story 2: Correct State Machine Enforcement — AC: completed task submit returns error "task already completed, create a subtask if re-work needed"
- Story 1: Safe Concurrent Task Operations — AC: concurrent submit has lock conflict error

## Hard Rules

- `--force` must be completely removed, not hidden/deprecated.
- BlockedReason format: `"auto-downgrade: testsFailed=N"` (exact format from design).

## Implementation Notes

Follow the caller pattern from design Interface 1. The auto-downgrade path (tests failed) also needs ValidateTransition with RoleSubmit + blocked target, and must set BlockedReason with the count.
