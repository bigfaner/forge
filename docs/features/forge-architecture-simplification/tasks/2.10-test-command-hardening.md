# 2.10: Test command hardening

## Description

Harden test commands: `test promote` rejects path traversal (`../`), `test verify` returns errors on parse failure (not zero-value) and marks unverifiable contracts (not OK).

## Reference Files

- design/tech-design.md — PRD Coverage Map § EC-6, EC-7, EC-8
- design/tech-design.md — Error Handling § per-file migration (test_promote.go, test_verify.go rows)

## Acceptance Criteria

- [ ] `test_promote.go`: `../` in journey name → `ErrInvalidPath` AIError (exit 2)
- [ ] `test_verify.go`: parse failure → `ErrEvalParseFailure` error (exit 1), not silent zero-value
- [ ] `test_verify.go`: no Fact Table → `ErrContractUnverifiable` (exit 2), not "OK"
- [ ] `os.Exit(1)` calls replaced with AIError + Exit()
- [ ] `go build ./...` + tests pass

## User Stories

- Story 3: Actionable Error Messages — AC: consistent error format across all commands

## Hard Rules

- Path traversal check uses `filepath.Base()` + reject `..` components. Do NOT use regex.

## Implementation Notes

See design Error Handling for the exit code mapping: ErrInvalidPath → exit 2 (blocking), ErrEvalParseFailure → exit 1 (retryable), ErrContractUnverifiable → exit 2 (blocking).
