# 2.8: Quality gate fixes + error handling

## Description

Fix quality gate: use real task ID for SourceTaskID (not sentinel), count active-only fix-tasks (not lifetime), return error when no feature configured. Migrate quality_gate.go and worktree.go error output to AIError + Exit() pattern.

## Reference Files

- design/tech-design.md — BC-5, BC-6, BC-7, EC-1, EC-3
- `forge-cli/internal/cmd/quality_gate.go` — sentinel SourceTaskID, lifetime cap, nil return on infra failure
- `forge-cli/internal/cmd/worktree.go` — fmt.Errorf usage

## Acceptance Criteria

- [ ] Fix-task SourceTaskID uses actual blocked task ID (not `"quality-gate:N"` sentinel)
- [ ] Fix-task cap counts active (non-completed/rejected) tasks only
- [ ] No feature → returns `ErrFeatureNotSet` AIError (exit 1), not silent exit 0
- [ ] `quality_gate.go`: infrastructure errors returned (not `nil`)
- [ ] `worktree.go`: all `fmt.Errorf` replaced with AIError factory functions + Exit()
- [ ] Characterization tests updated for quality gate changes
- [ ] `go build ./...` + tests pass

## Hard Rules

- The sentinel `"quality-gate:N"` pattern must be completely eliminated

## Implementation Notes

For SourceTaskID, extract the blocked task's actual ID before creating the fix-task. For active-only cap, filter fix-tasks by status (exclude completed/rejected/skipped) before counting. For worktree.go, replace each `fmt.Errorf` with the appropriate AIError factory function.
