# 1.1: Delete dead code (unused exports)

## Description

Delete exported functions/variables that have zero references across `forge-cli/`. Use `grep -r` or Go tooling to confirm zero usage before deletion. Zero behavior change.

## Reference Files

- design/tech-design.md — CH-2: Dead code deletion

## Acceptance Criteria

- [ ] All unused exported functions identified via `grep -r` across `forge-cli/`
- [ ] Each candidate verified: no references in `internal/`, `pkg/`, `cmd/`
- [ ] Deleted functions listed in commit message for traceability
- [ ] `go build ./...` passes after each deletion
- [ ] Characterization tests (0.1) still pass unchanged
- [ ] No behavior change — pure deletion

## Hard Rules

- MUST NOT change any function signatures or behavior — only delete verified-dead code
- MUST NOT delete test files or test helpers
- Verify each deletion independently (delete → build → test → commit pattern)

## Implementation Notes

Focus on `pkg/` exports first (these affect public API). Check `internal/cmd/` second. Use `go build` as the compile-time safety net.
