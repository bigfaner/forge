# 3.4: Schema version + enum alignment

## Description

Add version field to ForgeConfig schema for forward-compatible field migration. Align project type enums between schema definition and CLI (fullstack/mobile/library/mixed).

## Reference Files

- design/tech-design.md — PRD Coverage Map § CE-4, CE-5

## Acceptance Criteria

- [ ] ForgeConfig has `Version` field (default "1")
- [ ] Schema enum and CLI enum are identical: fullstack, mobile, library, mixed
- [ ] Existing configs without version field treated as version "1"
- [ ] `go build ./...` + tests pass

## User Stories

No direct user story mapping.

## Hard Rules

- Version field must be backward-compatible — existing configs without it must still load.

## Implementation Notes

Add `Version string \`yaml:"version"\`` to ForgeConfig. Set default "1" when loading config without version. Sync enum values between the schema YAML definition and the Go const block.
