# 2.6: Migrate index writers to WithLock pattern

## Description

Migrate claim, add, build_index, and migrate commands from direct `os.WriteFile` / `SaveIndex` to the `WithLock + SaveIndexAtomic` pattern. Ensures all index writers use advisory locking and atomic writes.

## Reference Files

- design/tech-design.md — Interface 2 § Standard pattern for all index writers
- design/tech-design.md — PRD Coverage Map § DR-1, DR-2

## Acceptance Criteria

- [ ] `cmd/claim.go` uses `WithLock(indexPath, fn)` for index writes
- [ ] `cmd/add.go` uses `WithLock(indexPath, fn)` for index writes
- [ ] `pkg/task/build_index.go` uses `AtomicWrite` for index saves
- [ ] `cmd/migrate.go` (if exists) uses `WithLock` pattern
- [ ] No remaining calls to `os.WriteFile` on index.json paths (grep verified)
- [ ] `go build ./...` + `go test ./...` pass

## User Stories

- Story 1: Safe Concurrent Task Operations — AC: concurrent claim has lock conflict, no corrupted JSON

## Hard Rules

- Every index write path MUST go through WithLock. No exceptions.
- grep for `os.WriteFile` on any index path to verify complete migration.

## Implementation Notes

Follow the standard pattern from design Interface 2. Each command wraps its read-mutate-write cycle in `WithLock`. The submit.go migration is handled separately in task 2.4.
