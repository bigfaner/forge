# 3.1: RunE migration + Args validation

## Description

Migrate 35 commands from `Run` to `RunE` for consistent error propagation. Add `.Args` validation (cobra.NoArgs / MaximumNArgs) to 12 commands currently missing it. Unify output with PrintBlock.

## Reference Files

- design/tech-design.md — PRD Coverage Map § CC-1, CC-3, CC-4

## Acceptance Criteria

- [ ] All 35 `Run:` commands converted to `RunE:` returning `error`
- [ ] 12 commands with missing `.Args` have appropriate args validators
- [ ] All command output uses PrintBlock formatting
- [ ] No `os.Exit()` calls in command functions (all use Exit() or return error)
- [ ] `go build ./...` + `go test ./...` pass

## User Stories

- Story 3: Actionable Error Messages — AC: error format consistent across all commands

## Hard Rules

- Do NOT change command behavior during migration — only error propagation mechanism.

## Implementation Notes

Mechanical refactoring. For each command: change `Run: func(cmd *cobra.Command, args []string) { ... }` to `RunE: func(cmd *cobra.Command, args []string) error { ... }`. Replace `os.Exit()` with `return error`. Add `Args: cobra.NoArgs` or `Args: cobra.MaximumNArgs(1)` as appropriate.
