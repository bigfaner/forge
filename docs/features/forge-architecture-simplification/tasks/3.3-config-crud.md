# 3.3: Config set/get CRUD

## Description

Implement `SetAutoKey` and `GetAutoKeyValue` in `pkg/profile/config.go` for all 4 auto fields. Wire to `forge config set <key> <value>` and `forge config get <key>` commands.

## Reference Files

- design/tech-design.md — Interface 4: Config Set/Get (signatures, valid keys, error behavior)

## Acceptance Criteria

- [ ] `SetAutoKey(projectRoot, key, value string) error` — sets key, returns ErrInvalidInput for unknown keys
- [ ] `GetAutoKeyValue(projectRoot, key string) (string, bool, error)` — gets key value
- [ ] Valid keys: `e2eTest`, `consolidateSpecs`, `cleanCode`, `gitPush`
- [ ] `forge config set auto.cleanCode true` works end-to-end
- [ ] `forge config get auto.e2eTest` works end-to-end
- [ ] Unknown key returns meaningful error
- [ ] `go build ./...` + tests pass

## User Stories

- Story 5: Complete Configuration Control — all 4 ACs: set/get works, init covers all fields, missing config error

## Hard Rules

- Use `gopkg.in/yaml.v3` for config parsing (not hand-written parser).

## Implementation Notes

See design Interface 4 for exact signatures and error behavior. The config file is `.forge/config.yaml` relative to project root.
