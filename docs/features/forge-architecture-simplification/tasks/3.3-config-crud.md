# 3.3: Config set/get CRUD (7 auto keys)

## Description

Implement `SetAutoKey` and `GetAutoKeyValue` in `pkg/forgeconfig/config.go` for all 7 auto fields. Wire to `forge config set <key> <value>` and `forge config get <key>` commands.

## Reference Files

- design/tech-design.md — Interface 4: Config Set/Get (signatures, valid keys, error behavior)
- forge-cli/pkg/forgeconfig/config.go — AutoConfig struct with 7 fields

## Acceptance Criteria

- [ ] `SetAutoKey(projectRoot, key, value string) error` — sets key, returns ErrInvalidInput for unknown keys
- [ ] `GetAutoKeyValue(projectRoot, key string) (string, bool, error)` — gets key value
- [ ] Valid keys: `e2eTest`, `consolidateSpecs`, `cleanCode`, `validation`, `runTasks`, `gitPush`, `knowledgeSave`
- [ ] ModeToggle fields (e2eTest–knowledgeSave except gitPush) accept `quick:true/false`, `full:true/false` sub-values
- [ ] `gitPush` is plain bool — accepts `true`/`false`
- [ ] `forge config set auto.cleanCode true` works end-to-end
- [ ] `forge config get auto.e2eTest` works end-to-end
- [ ] Unknown key returns meaningful error
- [ ] `go build ./...` + tests pass

## User Stories

- Story 5: Complete Configuration Control — all 4 ACs: set/get works, init covers all fields, missing config error

## Hard Rules

- Use `gopkg.in/yaml.v3` for config parsing (not hand-written parser).

## Implementation Notes

See design Interface 4 for exact signatures and error behavior. The config file is `.forge/config.yaml` relative to project root. The `pkg/forgeconfig/` package already defines the AutoConfig struct with ModeToggle type — reuse it for type-safe key validation.
