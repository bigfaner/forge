# 3.3: Config set/get CRUD

## Description

Add `SetConfigValue` to `pkg/forgeconfig/config.go` for single-key config updates. Wire to `forge config set <key> <value>` CLI command. `GetConfigValue` already exists with dot-notation support.

## Reference Files

- design/tech-design.md — Interface 4: Config Set/Get
- `forge-cli/pkg/forgeconfig/config.go` — existing GetConfigValue, WriteConfig
- `forge-cli/internal/cmd/config.go` — existing config get command

## Acceptance Criteria

- [ ] `SetConfigValue(projectRoot, key, value string) error` in `pkg/forgeconfig/`
- [ ] `forge config set <key> <value>` CLI command registered
- [ ] Supports all auto keys: `auto.e2eTest.quick`, `auto.gitPush`, `worktree.source-branch`, etc.
- [ ] Unknown key returns meaningful error
- [ ] End-to-end: `forge config set auto.cleanCode true` → `forge config get auto.cleanCode` → `true`
- [ ] `go build ./...` + tests pass

## Hard Rules

- Use `gopkg.in/yaml.v3` for config parsing (already used by forgeconfig)

## Implementation Notes

`GetConfigValue` already supports dot-notation reading. `SetConfigValue` needs to read config → find key by dot path → set value → write config. The AutoConfig struct already defines ModeToggle type for quick/full sub-fields.
