# 1.2: init-justfile backend template (15 recipes)

## Description

Create 15 recipe string literals for pure backend projects (Go/Rust/Python) in the init-justfile command file. These recipes have no scope parameters — each recipe directly calls the language toolchain command. The Go toolchain is the primary target (forge is a Go project).

## Reference Files

- `plugins/forge/commands/init-justfile.md` — init-justfile command file (target for template insertion)
- `docs/features/justfile-standard-vocabulary/design/tech-design.md` — Model 5: Recipe Templates (backend column)
- `docs/features/justfile-standard-vocabulary/prd/prd-spec.md` — Section 5.1: 标准命令词汇表

## Acceptance Criteria

- [ ] All 15 commands defined: project-type, compile, build, run, dev, test, test-e2e, lint, fmt, check, clean, install, ci, e2e-setup, e2e-verify
- [ ] No scope parameters on any recipe
- [ ] `project-type` outputs `@echo "backend"`
- [ ] Each recipe uses appropriate Go commands as defaults: `go vet ./...` (compile), `go build ./...` (build), `go test -race ./...` (test), etc.
- [ ] `e2e-setup`, `e2e-verify`, `test-e2e`, `ci` are shared templates (identical across project types)
- [ ] Templates stored as string literals ready for assembly by Task 2.1
- [ ] All recipes use `#!/usr/bin/env bash` shebang where multi-line

## User Stories

- **Story 2: 自适应项目初始化**
  - Given 一个包含 `go.mod` 但无 `package.json` 的项目, When 运行 `/init-justfile`, Then 生成的 justfile 无 scope 参数, `just project-type` 输出 `backend`

## Implementation Notes

- The template is a block of justfile syntax embedded in the init-justfile command file as a string literal
- Focus on Go toolchain commands; Rust/Python variants are future work (init-justfile can extend later)
- `e2e-setup` and `e2e-verify` are identical across all project types — define once, reuse
