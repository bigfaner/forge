# 1.3: init-justfile frontend template (15 recipes)

## Description

Create 15 recipe string literals for pure frontend projects (Node.js/npm) in the init-justfile command file. These recipes have no scope parameters — each recipe directly calls npm toolchain commands.

## Reference Files

- `plugins/forge/commands/init-justfile.md` — init-justfile command file (target for template insertion)
- `docs/features/justfile-standard-vocabulary/design/tech-design.md` — Model 5: Recipe Templates (frontend column)
- `docs/features/justfile-standard-vocabulary/prd/prd-spec.md` — Section 5.1: 标准命令词汇表

## Acceptance Criteria

- [ ] All 15 commands defined: project-type, compile, build, run, dev, test, test-e2e, lint, fmt, check, clean, install, ci, e2e-setup, e2e-verify
- [ ] No scope parameters on any recipe
- [ ] `project-type` outputs `@echo "frontend"`
- [ ] Each recipe uses appropriate npm commands: `npm run compile` (compile), `npm run build` (build), `npm test` (test), etc.
- [ ] `e2e-setup`, `e2e-verify`, `test-e2e`, `ci` are shared templates (identical across project types)
- [ ] Templates stored as string literals ready for assembly by Task 2.1

## User Stories

- **Story 2: 自适应项目初始化**
  - Given 一个包含 `package.json` 但无 `go.mod` 的项目, When 运行 `/init-justfile`, Then 生成的 justfile 无 scope 参数, `just project-type` 输出 `frontend`

## Implementation Notes

- The template is a block of justfile syntax embedded in the init-justfile command file as a string literal
- Assumes standard npm scripts exist (`compile`, `build`, `test`, `lint`, etc.)
- `e2e-setup` and `e2e-verify` are identical across all project types — define once, reuse
