# 1.5: Skill/agent/command migration + validation

## Description

Migrate 4 files from raw shell commands to standard vocabulary (`just compile` instead of `just build` for type-checking, `just run` instead of `npx serve`), then validate 10 additional files that already use standard commands and need no changes.

### Code Changes (4 files)

| # | File | Change |
|---|------|--------|
| 1 | `plugins/forge/skills/run-e2e-tests/SKILL.md` | Replace `npx serve` → `just run` |
| 2 | `plugins/forge/agents/execute-task.md` | Replace `just build && just test` → `just compile && just test` |
| 3 | `plugins/forge/agents/task-executor.md` | Replace `just build && just test` → `just compile && just test` |
| 4 | `plugins/forge/agents/error-fixer.md` | Replace `just build && just test` → `just compile && just test` |

Note: `breakdown-tasks/SKILL.md` scope annotation is a separate task (2.2) — this task does NOT modify it.

- [ ] `execute-task.md` contains `just compile && just test` and does NOT contain `just build && just test`
- [ ] `task-executor.md` contains `just compile && just test` and does NOT contain `just build && just test`
- [ ] `error-fixer.md` contains `just compile && just test` and does NOT contain `just build && just test`

- [ ] No file contains raw toolchain commands (`go test`, `npm run build`, `npx serve`) unless in documentation/comments explaining the standard vocabulary

## User Stories

- **Story 1: 标准化命令执行**
  - Given 一个使用 forge 插件的项目, When 任意 skill 执行构建/测试/编译操作时, Then 通过 `just <verb>` 调用, 不直接调用 `go`、`npm`、`cargo` 等语言工具链

## Implementation Notes

- Use sed-like replacements — these are mechanical text changes, not logic changes
- The `just build` → `just compile` replacement is because "build" in the new vocabulary means full compilation + packaging, while "compile" is type-checking + transpilation for fast feedback
- Validate that no unintended replacements occur (e.g., `just build-frontend` should NOT become `just compile-frontend`)
