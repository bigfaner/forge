# 3.1: Update forge project justfile as mixed reference

## Description

Update the forge project's existing justfile to use the new 15-command standard vocabulary as a mixed project reference implementation. Add scope parameters to applicable recipes, add project-type recipe, and ensure all 15 standard commands work correctly.

## Reference Files

- `justfile` (project root) — Target file to update
- `docs/features/justfile-standard-vocabulary/design/tech-design.md` — Model 5: mixed project template
- `tests/e2e/justfile-e2e-integration/cli.spec.ts` — Existing e2e tests (20/20 passing, must remain passing)

## Acceptance Criteria

- [ ] All 15 standard commands present in justfile
- [ ] `project-type` recipe outputs `mixed`
- [ ] 10 scoped recipes use bash case dispatch (compile, build, run, dev, test, lint, fmt, check, clean, install)
- [ ] 5 unscoped recipes present (test-e2e, ci, e2e-setup, e2e-verify, project-type)
- [ ] Boundary markers present: `# --- forge standard recipes ---` / `# --- end forge standard recipes ---`
- [ ] Existing e2e tests pass: `just test-e2e --feature justfile-e2e-integration`
- [ ] `just project-type` returns `mixed`
- [ ] `just compile frontend` executes frontend compile only
- [ ] `just compile backend` executes backend compile only
- [ ] `just compile` executes both frontend and backend compile

## User Stories

- **Story 1: 标准化命令执行**
  - Given 一个使用 forge 插件的项目, When 任意 skill 执行构建/测试/编译操作时, Then 通过 `just <verb>` 调用

## Implementation Notes

- Forge is itself a mixed project (Go backend + Node.js frontend in the e2e test area)
- Existing recipes may have custom logic — preserve within boundary markers, add new standard recipes
- This is the reference implementation that other mixed projects will be compared against
- Run existing e2e tests after update to verify no regressions
