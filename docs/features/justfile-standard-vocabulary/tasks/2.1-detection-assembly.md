# 2.1: init-justfile project-type detection + template assembly

## Description

Implement the project-type detection logic (Model 4: ProjectDetection) in the init-justfile command file. Detect project marker files (`package.json`, `go.mod`, `Cargo.toml`, `pyproject.toml`), classify the project as frontend/backend/mixed, and assemble the final justfile by selecting the correct template from tasks 1.2/1.3/1.4. Also implement boundary-marker merge for existing justfiles.

## Reference Files

- `plugins/forge/commands/init-justfile.md` — Target file for all changes
- `docs/features/justfile-standard-vocabulary/design/tech-design.md` — Model 4 (ProjectDetection), Interface 2 (project-type Recipe), Security Considerations (覆盖保护)
- `docs/features/justfile-standard-vocabulary/prd/prd-spec.md` — Section 5.2: 自适应 Justfile 生成

## Acceptance Criteria

- [ ] Detection logic checks for `package.json` (frontend signal), `go.mod`/`Cargo.toml`/`pyproject.toml` (backend signals)
- [ ] Classification: both signals → `mixed`, frontend only → `frontend`, backend only → `backend`, neither → error message
- [ ] Generates correct `project-type` recipe variant for each project type
- [ ] Selects and assembles the matching template (backend/frontend/mixed) from tasks 1.2-1.4
- [ ] Boundary marker merge: when justfile exists with `# --- forge standard recipes ---` markers, replaces only the marked section
- [ ] `--force` flag support: skips user confirmation for agent use (non-interactive)
- [ ] Interactive confirmation: when no boundary markers exist and justfile exists, prompts user before overwriting (human users only)
- [ ] Generated justfile wrapped in boundary markers: `# --- forge standard recipes ---` / `# --- end forge standard recipes ---`
- [ ] Error message when no known project markers detected

## User Stories

- **Story 2: 自适应项目初始化**
  - Given 一个包含 `package.json` 但无 `go.mod` 的项目, When 运行 `/init-justfile`, Then 生成的 justfile 无 scope 参数, `just project-type` 输出 `frontend`
  - Given 一个包含 `go.mod` 但无 `package.json` 的项目, When 运行 `/init-justfile`, Then 生成的 justfile 无 scope 参数, `just project-type` 输出 `backend`
  - Given 一个同时包含 `package.json` 和 `go.mod` 的项目, When 运行 `/init-justfile`, Then 生成的 justfile 支持 scope 参数, `just project-type` 输出 `mixed`

## Implementation Notes

- This is a prompt engineering task — the init-justfile command file instructs Claude to generate the justfile
- The detection logic is expressed as natural language instructions that Claude follows when running the command
- Boundary markers enable idempotent re-runs without losing user custom recipes
- The `--force` flag is critical for agent workflows (PRD requirement: no interactive prompts for agents)
