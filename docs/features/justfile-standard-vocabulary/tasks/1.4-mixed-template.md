# 1.4: init-justfile mixed template (15 recipes with scope)

## Description

Create 15 recipe string literals for mixed projects (frontend + backend) in the init-justfile command file. 10 recipes include `scope=""` position parameter with bash case dispatch. 5 recipes (project-type, test-e2e, ci, e2e-setup, e2e-verify) have no scope parameter.

## Reference Files

- `plugins/forge/commands/init-justfile.md` — init-justfile command file (target for template insertion)
- `docs/features/justfile-standard-vocabulary/design/tech-design.md` — Interface 1 (Recipe Contract), Model 5 (mixed column)
- `docs/features/justfile-standard-vocabulary/prd/prd-spec.md` — Section 5.1: scope 参数规则

## Acceptance Criteria

- [ ] 10 scoped recipes with bash case dispatch: compile, build, run, dev, test, lint, fmt, check, clean, install
- [ ] 5 unscoped recipes: project-type, test-e2e, ci, e2e-setup, e2e-verify
- [ ] `project-type` outputs `@echo "mixed"`
- [ ] Each scoped recipe follows Interface 1 pattern: `scope=""` parameter, bash case with `frontend)`, `backend)`, `""`, `*)` branches
- [ ] `*)` branch: `echo "[forge] invalid scope '{{scope}}'; expected frontend/backend" >&2; exit 1`
- [ ] `""` branch: executes both frontend and backend commands
- [ ] All bash recipes use `set -euo pipefail`
- [ ] Frontend commands use npm toolchain, backend commands use Go toolchain
- [ ] Templates stored as string literals ready for assembly by Task 2.1

## User Stories

- **Story 1: 标准化命令执行**
  - Given 一个混合项目, When 开发者执行 `just build frontend` 时, Then 仅执行前端构建
- **Story 4: Agent 友好的命令输出**
  - Given 任意 `just <verb>` 命令, When 执行成功时, Then 退出码为 0
  - Given 任意 `just <verb>` 命令, When 执行失败时, Then 退出码非 0, stderr 包含错误信息
  - Given agent 连续执行 `just install`、`just compile`、`just test`, When 全部成功时, Then 每步退出码均为 0

## Implementation Notes

- The bash case pattern is the core design decision — see tech-design "Alternatives Considered" for rationale
- Error output must go to stderr (`>&2`) for agent-friendly separation
- The `""` empty scope means "run all" — both frontend and backend commands chained with `&&`
- `e2e-setup`, `e2e-verify`, `test-e2e`, `ci` are shared templates identical to tasks 1.2 and 1.3
