# 3.1: Migrate existing conventions to new schema

## Description

将现有 3 个 Convention 文件（go.md、ginkgo.md、vitest.md）迁移到新的 4-section schema（framework、discovery、structure、assertions），并移动到 `docs/conventions/testing/` 子目录。直接迁移，不保留旧 schema 兼容。

## Reference Files

- `docs/features/test-capability-v2/prd/prd-spec.md` § Scope (Convention 迁移), § Compatibility Requirements
- `docs/features/test-capability-v2/design/tech-design.md` § Interface 2 (Convention Files), § PRD Coverage Map Phase 3 (Convention 迁移)
- `docs/conventions/testing-go.md` — 现有，需迁移
- `docs/conventions/testing-ginkgo.md` — 现有，需迁移
- `docs/conventions/testing-vitest.md` — 现有，需迁移
- `docs/conventions/forge-distribution.md` — 分发模型约束

## Acceptance Criteria

- [ ] 3 个现有 Convention 文件迁移到 `docs/conventions/testing/` 子目录：
  - `testing-go.md` → `testing/go.md`
  - `testing-ginkgo.md` → `testing/ginkgo.md`
  - `testing-vitest.md` → `testing/vitest.md`
- [ ] 每个迁移后的文件包含完整的 4 个 section：
  - `framework`：name, version, language, runner_command
  - `discovery`：test_dir, file_pattern, exclude_pattern
  - `structure`：suite_pattern, case_pattern, hook_pattern
  - `assertions`：style (assert/expect/should), custom_matchers
- [ ] 迁移后的文件通过 Convention schema 验证（4-section 完整性检查）
- [ ] 旧位置的文件已删除
- [ ] 引用旧路径的 skill rules 已更新为新路径
- [ ] 现有 Go/Vitest/Ginkgo 项目跑完整管线无报错（回归验证）

## User Stories

- **Story 5** (prd-user-stories.md): 通用性 — Convention 文件更新为新 schema

## Hard Rules

- 直接迁移，不保留旧 schema 兼容
- Convention 扩展不应破坏已有 Convention 的加载逻辑
- 迁移后 Convention 文件不使用 `domains` frontmatter 过滤

- 检查是否有其他文件引用旧的 Convention 路径（如 gen-test-scripts 的 rules）
