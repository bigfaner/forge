# 2.3: Create eval-contract rubric + command

## Description

创建 eval-contract 评测技能，用于评估 gen-contracts 生成的 Contract 文档质量。与 eval-journey 共享评分维度框架和阈值体系，但评估对象和上下文不同。

## Reference Files

- `docs/features/test-capability-v2/prd/prd-spec.md` § Scope In Scope 第 4、6 项, § Eval Rubric 评分维度框架
- `docs/features/test-capability-v2/design/tech-design.md` § Interface 4, § Quality Gate Integration
- `plugins/forge/skills/eval/SKILL.md` — eval 框架
- `docs/conventions/forge-distribution.md` — 路径规则

## Acceptance Criteria

- [ ] Rubric 文件 `plugins/forge/skills/eval/rubrics/contract.md` 创建，包含：
  - Frontmatter: scale=1000, target=850, iterations=3, type=contract
  - 与 journey rubric 相同的 6 维度评分框架和最低阈值
  - Contract 特定评估重点：6 维度结构完整性、Outcome Preconditions 互斥性
- [ ] 命令文件 `plugins/forge/commands/eval-contract.md` 创建
- [ ] 同样的门禁逻辑：总分 ≥850 + 每维度不低于最低阈值 → 通过
- [ ] 3 轮迭代耗尽后 PAUSE_C（与 PAUSE_J 对称的恢复路径）
- [ ] LLM 解析失败处理与 eval-journey 一致
- [ ] 评分结果写入 `.forge/session.yaml` 的 eval_result 字段（覆盖上一轮）

## User Stories

- **Story 4** (prd-user-stories.md): 评测门禁 — Contract 文档生成后自动验证质量

## Hard Rules

- 与 eval-journey 共享评分维度定义，差异仅在评估对象（Journey vs Contract）
- Contract 评估额外关注：6 维度结构完整性 + Outcome Preconditions 互斥性
- 复用现有 eval 框架

## Implementation Notes

- 可考虑将共享维度定义提取为独立文件，journey.md 和 contract.md 引用。但如果 eval 框架不支持文件引用，则各自完整定义
- PAUSE_C 恢复路径与 PAUSE_J 对称：a. 跳过门禁继续 b. 放弃管线 c. 修改后重跑
