## Conventions Established

- **Surface rule pattern**: Each surface type gets a single surface-<type>.md file with 4 sections (Detection Signals, Testing Principles, Strategy Guidance, Required Outcome Reference). Extensibility requires only a new file, no pipeline changes.
- **Rule extraction for >5 lines**: Logic exceeding 5 lines is extracted to rules/*.md files, keeping SKILL.md under 350 lines.
- **Eval rubric uniformity**: Journey and contract rubrics share the same 6-dimension scoring framework and thresholds (total >= 850, per-dimension minimums).
- **Fact Table access pattern**: Go CLI provides structured read operations; LLM performs writes via CLI to guarantee correctness.
- **Confidence rating semantics**: HIGH >= 0.80, MEDIUM 0.40-0.80, LOW < 0.40. eval_skipped/eval_bypassed force LOW. LOW never blocks, only marks for REVIEW.
- **Best-effort surfaces**: Mobile surface uses Maestro YAML skeletons with manual-only marking for complex scenarios; env checks are non-blocking.
- **Quality gate serial dependency**: eval-journey -> eval-contract -> confidence rating, each stage independent pass/fail determination.
