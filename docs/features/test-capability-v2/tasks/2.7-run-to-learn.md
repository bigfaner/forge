# 2.7: Implement Run-to-Learn mechanism

## Description

实现 Run-to-Learn 机制：gen-test-scripts 生成骨架测试（有 setup + 执行 + 输出捕获，但无断言）→ 运行捕获实际输出 → 丰富 Fact Table → 重新生成更精确的测试。退出条件：≤3 轮或 Fact Table 覆盖率 ≥80%。

## Acceptance Criteria

- [ ] 新增规则文件 `plugins/forge/skills/gen-test-scripts/rules/run-to-learn.md`，定义 R2L 流程：
  - 生成骨架测试（setup + 执行 + 输出捕获，无断言）
  - 运行骨架测试，捕获 stdout/stderr/exit code
  - 将运行时数据写入 Fact Table（source: runtime, confidence: confirmed）
  - 重新生成更精确的测试
- [ ] 退出条件：≤3 轮，或 Fact Table 覆盖率 ≥80%
- [ ] 骨架测试有超时保护（默认 60s，可通过 `.forge/config.yaml` 配置）
- [ ] 失败处理（PRD § Run-to-Learn Failure Handling）：
  - 编译失败：记录到 Fact Table（kind: compilation_error），跳过本轮
  - 运行时崩溃：记录到 Fact Table（kind: runtime_crash），标记相关 Outcome 为 LOW
  - 脏数据输出：丢弃本轮数据，保留上一轮状态
  - API 写操作副作用：仅 GET 请求，写操作生成回滚语句
- [ ] 兜底原则：任何 R2L 失败不阻塞管线，降级使用静态信息
- [ ] R2L 通过 `.forge/config.yaml` 的 `run_to_learn: true` 或 CLI 标志启用

## User Stories

- **Story 6** (prd-user-stories.md): 信息增强 — Run-to-Learn 迭代提升测试精度

## Hard Rules

- Runtime fact 替换同一 subject+kind 的 static fact；若 runtime confidence 非 confirmed，static 保留为 fallback
- 覆盖率计算：`confirmed/runtime 事实覆盖的 Outcome 数 / 总 Outcome 数 × 100%`
- 骨架测试在临时目录运行，写操作使用隔离环境
- 环境未就绪时跳过 R2L，直接使用静态信息

## Implementation Notes

- RunToLearnConfig 数据模型：enabled, max_iterations(3), coverage_threshold(0.80), timeout_per_test(60s), skip_on_env_failure(true)
- R2L 依赖 Fact Table CLI（2.4）进行结构化写入
- 骨架测试生成依赖 gen-test-scripts 的 surface 差异化能力（2.6）
- 环境就绪检测由 2.8 负责，R2L 在运行前检查环境状态
