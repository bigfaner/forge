# 3.4: Inline testing metadata into skill rules

## Description

将 3 个独立的 testing 元数据 Convention 文件（testing-conventions.md、testing-journey-contract.md、testing-isolation.md）的内容内联到各 skill 的 rules 中，然后删除这些独立文件。减少文件数量，消除无人消费的纯文档。

## Reference Files

- `docs/features/test-capability-v2/design/tech-design.md` § PRD Coverage Map Phase 3 (Testing 元数据内联), § Appendix § Design Principles
- `docs/conventions/testing-conventions.md` — 需删除，内容内联
- `docs/conventions/testing-journey-contract.md` — 需删除，内容内联
- `docs/conventions/testing-isolation.md` — 需删除，内容内联
- `docs/conventions/forge-distribution.md` — 分发模型约束

## Acceptance Criteria

- [ ] `testing-conventions.md` 的内容已内联到相关 skill rules 中
- [ ] `testing-journey-contract.md` 的内容已内联到相关 skill rules 中
- [ ] `testing-isolation.md` 的内容已内联到相关 skill rules 中
- [ ] 3 个文件已删除
- [ ] 内联后的 skill rules 包含原有元数据的完整信息
- [ ] 管线功能不受影响（回归验证）

## User Stories

No direct user story mapping. 此任务为代码 hygiene 清理，消除独立的 testing 元数据文件。

## Hard Rules

- 内联时保留完整语义，不压缩或省略
- 确定每个文件的内容应内联到哪些 skill rules 中：
  - testing-conventions.md → gen-test-scripts rules（Convention 使用指导）
  - testing-journey-contract.md → gen-journeys/gen-contracts rules（Journey-Contract 元数据）
  - testing-isolation.md → run-tests rules（测试隔离策略）

## Implementation Notes

- 先读取 3 个文件的内容，确定每个 section 应内联到哪个 skill rule
- 内联方式：在对应 skill 的 rules 文件中添加引用或直接包含内容
- 删除文件后确认无其他文件引用旧路径
