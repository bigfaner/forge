# 2.5: Enhance gen-contracts with risk-driven density

## Description

增强 gen-contracts 技能，基于 Journey 的 risk_level 字段差异化衍生 Outcome 数量，并自动衍生边界/异常 Outcome。依赖 2.1 创建的 surface rules（必须 Outcome 参考）和风险分级判定规则。

## Reference Files

- `docs/features/test-capability-v2/prd/prd-spec.md` § Flow Description 阶段二步骤 6, § Per-Surface Strategy (风险驱动测试密度), § Other Notes § Fact Table Core Model
- `docs/features/test-capability-v2/design/tech-design.md` § PRD Coverage Map (风险驱动密度, 必须 Outcome)
- `plugins/forge/skills/gen-contracts/SKILL.md` — 需增强
- `docs/conventions/forge-distribution.md` — 路径规则

## Acceptance Criteria

- [ ] 新增规则文件 `plugins/forge/skills/gen-contracts/rules/risk-density.md`，定义风险分级策略：
  - High：每 Step 3-5 个 Outcome（含必须边界），总测试数 13-20
  - Medium：每 Step 2-3 个 Outcome，总测试数 8-12
  - Low：每 Step 1-2 个 Outcome，总测试数 4-7
- [ ] gen-contracts SKILL.md 增强为：读取 Journey risk_level → 按密度规则衍生 Outcome
- [ ] 自动衍生边界/异常 Outcome：基于 surface 类型的 required_outcomes + 项目 Fact Table
- [ ] 合约生成后执行 schema 验证（6 维度结构完整性 + Outcome Preconditions 互斥性检查）
- [ ] Schema 验证失败时自动重试 1 次（将 schema 错误注入 prompt），重试仍失败暂停管线
- [ ] 静态 Fact Table（代码侦察结果）写入 `.forge/fact-table.json`（source: static）

## User Stories

- **Story 2** (prd-user-stories.md): 深度测试 — 风险驱动的边界/异常覆盖

## Hard Rules

- 必须 Outcome 衍生受 surface rule 的 required_outcomes 约束：API 每个认证端点衍生 unauthorized、TUI 每个异步 Cmd 衍生 timeout、CLI 衍生 not-found + already-exists、WebUI 衍生 validation-error + session-expired
- 同一 Step 内 Outcome 的 Preconditions 必须互斥
- LLM 衍生的边界 Outcome 需标注 `source: inferred` + 推理依据

## Implementation Notes

- 风险分级由 gen-journeys 在 Journey 中标记（risk_level 字段），gen-contracts 读取并应用密度规则
- 静态 Fact Table 由 gen-contracts 在生成合约时通过代码侦察生成，包含函数签名、输出格式、错误码等
- Schema 验证由 gen-contracts SKILL.md 在合约生成后立即执行
