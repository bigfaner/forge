# 3.3: Enhance test-guide with auto-detect + draft generation

## Description

增强 test-guide 技能，使其能自动扫描项目文件信号检测测试框架，并生成 Convention 草稿供用户审核。草稿包含完整的 4-section schema，用户审核修改量目标 ≤20%。

## Reference Files

- `docs/features/test-capability-v2/prd/prd-spec.md` § Flow Description 阶段一步骤 3, § Scope (test-guide 增强)
- `docs/features/test-capability-v2/design/tech-design.md` § PRD Coverage Map Phase 3 (test-guide)
- `plugins/forge/skills/test-guide/SKILL.md` — 需增强
- `docs/conventions/forge-distribution.md` — 路径规则

## Acceptance Criteria

- [ ] 新增规则文件 `plugins/forge/skills/test-guide/rules/signal-detection.md`，定义框架检测信号：
  - Go testing: go.mod + *_test.go + testing.T
  - Ginkgo: go.mod + ginkgo dependency + . + ginkgo import
  - Vitest: package.json + vitest dependency + *.test.ts
  - pytest: pyproject.toml/setup.py + pytest dependency + test_*.py
  - JUnit: pom.xml/build.gradle + JUnit dependency + *Test.java
  - Rust: Cargo.toml + #[cfg(test)] + tests/ 目录
- [ ] 新增规则文件 `plugins/forge/skills/test-guide/rules/draft-generation.md`，定义草稿生成规则：
  - 基于检测到的框架生成包含 4-section 的 Convention 草稿
  - 草稿通过 Convention schema 验证
  - 用户审核后确认或拒绝
- [ ] 用户拒绝草稿时，基于反馈重新生成（保留认可部分，仅修正被指出的错误），最多重试 2 次
- [ ] 2 次重试耗尽后输出最终草稿供用户手动编辑，管线暂停等待确认
- [ ] Convention 草稿写入 `docs/conventions/testing/` 目录

## User Stories

- **Story 5** (prd-user-stories.md): 通用性 — 全新项目首次运行时自动检测框架并生成 Convention 草稿

## Hard Rules

- 草稿需用户审核确认，不自动应用
- 生成的草稿必须通过 Convention schema 验证（4-section 完整性）
- 检测信号应排除误判（如 package.json 含 React 依赖应排除 CLI 判断）

## Implementation Notes

- 检测信号与 surface 检测信号不同：surface 检测判断"项目是什么类型"，test-guide 检测判断"项目用什么测试框架"
- 草稿生成可以参考内置 Convention 文件（3.2 创建的 pytest.md 等）作为模板
- 草稿修改量评估：`diff --stat` 统计用户修改行数占草稿总行数比例，目标 ≤20%（human-verified）
