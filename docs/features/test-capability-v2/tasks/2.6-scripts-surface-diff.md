# 2.6: Enhance gen-test-scripts for surface differentiation

## Description

增强 gen-test-scripts 技能，根据 surface 类型（CLI/TUI/WebUI/Mobile/API）差异化生成测试代码策略和层级组合。CLI 侧重 Contract 测试（subprocess 执行模型），WebUI/API 平衡 50/50，Mobile 生成 Maestro YAML 骨架。

## Reference Files

- `docs/features/test-capability-v2/prd/prd-spec.md` § Per-Surface Strategy, § Flow Description 阶段三步骤 8
- `docs/features/test-capability-v2/design/tech-design.md` § PRD Coverage Map (gen-test-scripts 增强)
- `plugins/forge/skills/gen-test-scripts/SKILL.md` — 需增强
- `docs/conventions/forge-distribution.md` — 路径规则

## Acceptance Criteria

- [ ] gen-test-scripts SKILL.md 增强为：读取 `.forge/config.yaml` surface 字段 → 按类型差异化生成策略
- [ ] 按 surface 类型创建类型特定规则（放入 skills/gen-test-scripts/ 的 rules 或 types 子目录）：
  - CLI：Contract 测试占比 ≥ 80%，subprocess 执行模型，自动检查二进制文件+环境变量隔离
  - TUI：Contract 测试占比 ≥ 80%，terminal I/O 测试
  - WebUI：平衡 50/50，Convention 定义的浏览器框架
  - API：平衡 50/50，HTTP client 测试
  - Mobile：Maestro YAML 骨架 + deep link 测试，复杂场景标记 manual-only
- [ ] 生成后执行语法/可执行性验证：(a) 测试文件语法正确 (b) 导入路径可解析
- [ ] 验证失败自动重试 1 次，重试仍失败标记 gen-failed 并跳过
- [ ] 测试代码写入 `tests/<journey>/` 目录，按 Convention discovery 规则可被发现

## User Stories

- **Story 3** (prd-user-stories.md): Surface 差异化 — 按 surface 类型定制测试策略

## Hard Rules

- Contract 测试占比公式：`Contract 测试函数 / (Contract 测试函数 + Journey 烟测试函数) × 100%`
- Mobile 为 best-effort：不要求完整覆盖，复杂场景标记 manual-only
- 生成策略由 surface rule 文件指导，Convention 文件提供框架知识，两者正交

## Implementation Notes

- surface 类型从 `.forge/config.yaml` 读取（由 2.1 的 gen-journeys 检测并写入）
- Convention 文件从 `docs/conventions/testing/` 加载（由 3.1/3.2 创建/迁移）
- 类型特定规则可以是独立文件或 SKILL.md 内的条件分支，视复杂度决定
