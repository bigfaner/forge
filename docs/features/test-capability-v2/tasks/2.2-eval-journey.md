# 2.2: Create eval-journey rubric + command

## Description

创建 eval-journey 评测技能，用于评估 gen-journeys 生成的 Journey 文档质量。复用现有 eval 框架（scorer-gate-revise 循环），新增 journey 类型的 rubric 文件和斜杠命令入口。

## Reference Files

- `docs/features/test-capability-v2/prd/prd-spec.md` § Scope In Scope 第 3、6 项, § Eval Rubric 评分维度框架, § Eval Gate Calibration
- `docs/features/test-capability-v2/design/tech-design.md` § Interface 4 (Eval Rubric Dimension), § Data Model 4 (EvalSession), § Quality Gate Integration
- `plugins/forge/skills/eval/SKILL.md` — eval 框架，理解 rubric 加载和 scorer-gate-revise 循环
- `plugins/forge/skills/eval/experts/` — 评分协议和专家角色
- `plugins/forge/commands/eval-test-cases.md` — 参考现有 eval 命令格式（将被删除但可作为模板）
- `docs/conventions/forge-distribution.md` — 路径规则

## Acceptance Criteria

- [ ] Rubric 文件 `plugins/forge/skills/eval/rubrics/journey.md` 创建，包含：
  - Frontmatter: scale=1000, target=850, iterations=3, type=journey
  - 6 个评分维度定义（完整性 200、语义纯度 200、前置条件互斥性 150、事实依据 150、Surface 适配 150、一致性 150）
  - 每维度最低阈值（完整性 ≥120、语义纯度 ≥120、前置条件互斥性 ≥90、事实依据 ≥90、Surface 适配 ≥90、一致性 ≥90）
  - 总分 ≥850 且每维度不低于最低阈值才通过
- [ ] 命令文件 `plugins/forge/commands/eval-journey.md` 创建，调用 eval 框架并传入 journey rubric
- [ ] Eval 失败时自动迭代修正（最多 3 轮），基于评分反馈针对性修正
- [ ] 3 轮后仍未达阈值，暂停管线输出评分+未通过项明细
- [ ] LLM 解析失败时重试 1 次，仍失败标记 eval-skipped + 置信度降为 LOW
- [ ] 评分结果写入 `.forge/session.yaml` 的 eval_result 字段

## User Stories

- **Story 4** (prd-user-stories.md): 评测门禁 — Journey 文档生成后自动验证质量

## Hard Rules

- 复用现有 eval 框架，不新建独立 skill
- Rubric 中事实依据维度需区分"事实声明"（需 fact_id 追溯）和"合理推理声明"（需 required_outcomes 规则支撑 + source: inferred 标注）
- Surface 适配维度根据检测到的 surface 类型动态适配：按该 surface 类型的 required_outcomes 检查覆盖率

## Implementation Notes

- eval 框架通过参数化 rubric 路径实现多类型支持，新 rubric 文件放入 `eval/rubrics/` 目录即可
- PAUSE_J 恢复路径（跳过门禁/放弃管线/修改后重跑）由 gen-journeys SKILL.md 编排，不属于 rubric 本身
- EvalSession 数据模型存储每轮评分结果，revise 步骤读取 failed_dims 进行针对性修正
