# 2.4: Implement forge fact CLI subcommand

## Description

在 Go CLI 层实现 `forge fact` 子命令，提供 Fact Table 的结构化读写操作。这是唯一需要 Go 代码变更的组件——LLM 直接操作 JSON 容易导致数据损坏，需要 CLI 保证更新语义（runtime→static 替换、fallback 保留）的正确性。

## Acceptance Criteria

- [ ] `forge fact list [--source static|runtime] [--confidence confirmed|inferred]` — 列表展示 fact 摘要，支持按 source 和 confidence 过滤
- [ ] `forge fact get <fact_id>` — 查看单条 fact 完整内容（value 原样 pretty-print）
- [ ] `forge fact summary` — 按 source/confidence/kind 统计，含覆盖率指标
- [ ] Fact Table 存储于 `.forge/fact-table.json`，结构为 FactEntry 数组
- [ ] FactEntry 包含：fact_id（格式 `{subject}-{kind}-{nonce}`）、source、subject、kind、value（自由格式 JSON）、confidence、updated_at
- [ ] Runtime fact 替换同一 subject+kind 的 static fact
- [ ] 若 runtime confidence 非 confirmed，static fact 保留为 fallback（不删除）
- [ ] JSON 损坏或格式无效时输出明确错误提示 + 恢复建议（exit code 1）
- [ ] 单元测试覆盖率 ≥ 90%（list/get/summary、runtime→static 替换、fallback 逻辑）

## User Stories

- **Story 6** (prd-user-stories.md): Run-to-Learn — 骨架测试捕获运行时行为写入 Fact Table

## Hard Rules

- `forge fact` 只提供读取子命令（list/get/summary）。写入操作（添加/更新 fact）由 gen-contracts（静态侦察）和 Run-to-Learn（运行时捕获）通过 LLM 调用 CLI 完成——需要在技能执行时由 LLM 生成 JSON 并通过 CLI 写入
- 事实替换语义必须正确：runtime confirmed 替换 static；runtime non-confirmed 保留 static 为 fallback
- 覆盖率计算：`confirmed/runtime 事实覆盖的 Outcome 数 / 总 Outcome 数 × 100%`

## Implementation Notes

- fact_id 的 nonce 部分用于避免碰撞，可使用时间戳或短随机串
- kind 枚举：signature、output_format、error_code、side_effect、precondition、compilation_error、runtime_crash
- confidence 枚举：confirmed、inferred、assumed
- value 为自由格式 JSON，结构随 kind 变化，CLI 不验证 value 内部结构
- `.forge/fact-table.json` 不存在时视为空表（自动创建空数组）
- 考虑是否需要 `forge fact add` 子命令供 LLM 写入，或写入由 LLM 直接操作 JSON 文件（但需要保证原子性）
