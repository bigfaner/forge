# 2.9: Add confidence rating system

## Description

为每个生成的测试标注置信度评级（HIGH/MEDIUM/LOW），基于 Fact Table 中 confirmed 事实占比计算。评级嵌入测试文件头部注释，并在测试报告中汇总。

## Reference Files

- `docs/features/test-capability-v2/prd/prd-spec.md` § Flow Description 阶段三步骤 11, § Other Notes § Fact Table Core Model (覆盖率计算)
- `docs/features/test-capability-v2/design/tech-design.md` § Data Model 2 (ConfidenceRating)
- `plugins/forge/skills/run-tests/SKILL.md` — 需增强
- `docs/conventions/forge-distribution.md` — 路径规则

## Acceptance Criteria

- [ ] 新增规则文件 `plugins/forge/skills/run-tests/rules/confidence.md`，定义评级规则：
  - HIGH：confirmed_fact_ratio ≥ 0.80
  - MEDIUM：0.40 ≤ confirmed_fact_ratio < 0.80
  - LOW：confirmed_fact_ratio < 0.40
- [ ] 置信度评级嵌入生成的测试文件头部注释（如 `// confidence: HIGH`）
- [ ] eval-skipped → 自动降为 LOW
- [ ] eval-bypassed → 自动降为 LOW
- [ ] LOW 置信度的测试仍执行，但在报告中标记为需人工审核（REVIEW）
- [ ] 测试报告包含：置信度分布统计 + VERIFY/REVIEW 标记统计
- [ ] ConfidenceRating 数据模型包含：level, confirmed_fact_ratio, total_outcomes, confirmed_outcomes, eval_skipped, eval_bypassed

## User Stories

- **Story 6** (prd-user-stories.md): 信息增强 — 生成的测试附带置信度评级

## Hard Rules

- confirmed_fact_ratio = Fact Table 中 `source=runtime` 且 `confidence=confirmed` 的事实覆盖的 Outcome 数 / 总 Outcome 数
- LOW 置信度不阻止测试执行，仅在报告中标记
- 用户可手动清除 eval-skipped 标记，清除后置信度由 Fact Table 覆盖率重新计算

## Implementation Notes

- 置信度计算依赖 Fact Table CLI（2.4）的 summary 命令获取覆盖率数据
- 评级逻辑在 run-tests 执行后计算，嵌入测试文件和报告中
