# 1.1: Retire gen-test-cases + eval-test-cases

## Description

删除 gen-test-cases 技能及其完整的评测能力（eval-test-cases 命令 + test-cases rubric + 5 个 per-type 子 rubric）。Journey-Contract 管线已完全覆盖其能力，双路径并存是用户困惑的主要来源。

## Acceptance Criteria

- [ ] `plugins/forge/skills/gen-test-cases/` 目录完全删除
- [ ] `plugins/forge/commands/eval-test-cases.md` 删除
- [ ] 以下 rubric 文件删除：
  - `plugins/forge/skills/eval/rubrics/test-cases.md`
  - `plugins/forge/skills/eval/rubrics/cli-test-cases.md`

- [ ] 全局搜索 `gen-test-cases`（排除 PRD/eval/history 文档）无匹配

## User Stories

- **Story 1** (prd-user-stories.md): 管线统一 — 用户只有一条清晰的测试生成路径

## Hard Rules

- 删除前确认无外部依赖引用（hooks、其他 skill 的 rules）
- eval SKILL.md 中的 rubric 加载逻辑不应硬编码 rubric 文件名（使用目录扫描或配置），确保删除 rubric 文件后 eval 框架正常工作

## Implementation Notes

- eval 框架（skills/eval/SKILL.md）通过参数指定 rubric 文件路径，不依赖固定 rubric 列表，删除 test-cases 系列 rubric 不会影响框架本身
- 如果 `forge task` 系统有 test.gen-cases 类型注册，需要从注册中移除
