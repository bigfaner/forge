# 3.2: Create Convention index + new conventions

## Description

创建 Convention 索引文件（index.md）和 3 个新增内置 Convention 文件（pytest、JUnit、Rust/cargo test）。索引文件列出所有 Convention 的名称、描述、适用条件，供 LLM 按需路由加载。

## Reference Files

- `docs/features/test-capability-v2/prd/prd-spec.md` § Goals (内置 ≥3 个新 Convention), § Scope (内置 Convention 扩充)
- `docs/features/test-capability-v2/design/tech-design.md` § Interface 2 (Convention Files), § PRD Coverage Map Phase 3
- `docs/conventions/testing/` — 3.1 迁移后的目录（go.md, ginkgo.md, vitest.md）
- `docs/conventions/forge-distribution.md` — 路径规则

## Acceptance Criteria

- [ ] `docs/conventions/testing/index.md` 创建，包含所有 Convention 的名称+描述+适用条件：
  - Go testing、Ginkgo、Vitest（3.1 迁移后的）
  - pytest、JUnit、Rust/cargo test（本任务新增的）
- [ ] `docs/conventions/testing/pytest.md` 创建，包含完整的 4-section schema：
  - framework: pytest, language: Python, runner_command: `pytest`
  - discovery: test_dir=tests/, file_pattern=test_*.py
  - structure: 函数式或 class-based
  - assertions: assert 语句 + pytest 丰富的比较
- [ ] `docs/conventions/testing/junit.md` 创建，包含完整的 4-section schema：
  - framework: JUnit 5, language: Java, runner_command: `mvn test` / `gradle test`
  - discovery: test_dir=src/test/java/, file_pattern=*Test.java
  - structure: @Test 注解方法
  - assertions: Assertions 类 (assertEquals, assertTrue 等)
- [ ] `docs/conventions/testing/rust.md` 创建，包含完整的 4-section schema：
  - framework: cargo test, language: Rust, runner_command: `cargo test`
  - discovery: tests/ 目录 + 内联 #[test] 函数
  - structure: #[test] 函数 + #[cfg(test)] mod
  - assertions: assert!, assert_eq!, assert_ne!
- [ ] 所有新文件通过 Convention schema 验证
- [ ] index.md 不使用 `domains` frontmatter，LLM 读取索引后自行判断相关性

## User Stories

- **Story 5** (prd-user-stories.md): 通用性 — 新项目快速接入测试

## Hard Rules

- 每个 Convention 文件必须包含 4 个 section（framework, discovery, structure, assertions），缺少任一视为无效
- Convention 与 Surface 正交：Convention 不知道 surface 的存在
- 不使用 `domains` frontmatter 过滤

## Implementation Notes

- Convention 文件是纯 markdown，供 LLM 按需读取
- 加载机制：两级加载——先读 index.md（索引），再根据项目上下文加载对应的 .md 文件
- 新增 Convention 只需添加 .md 文件 + 更新 index.md，无需修改管线代码
