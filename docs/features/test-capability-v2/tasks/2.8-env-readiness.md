# 2.8: Add environment readiness detection

## Description

增强 run-tests 技能，在测试执行前按 surface 类型检测执行环境是否就绪。不同 surface 类型有不同的检测关注点：CLI 检查二进制编译、TUI 检查二进制+stdin pipe、WebUI 检查 dev server、API 检查服务+DB。

## Reference Files

- `docs/features/test-capability-v2/prd/prd-spec.md` § Flow Description 阶段三步骤 10
- `docs/features/test-capability-v2/design/tech-design.md` § PRD Coverage Map (环境就绪检测)
- `plugins/forge/skills/run-tests/SKILL.md` — 需增强
- `docs/conventions/forge-distribution.md` — 路径规则

## Acceptance Criteria

- [ ] 新增规则文件 `plugins/forge/skills/run-tests/rules/env-check.md`，定义 per-surface 检测项：
  - CLI：二进制文件已编译
  - TUI：二进制文件已编译 + stdin pipe 可用
  - WebUI：dev server 可访问
  - API：服务可访问 + DB 连接可用
  - Mobile：Maestro CLI 已安装（可选，未安装不阻塞）
- [ ] run-tests SKILL.md 增强：测试执行前读取 surface 类型 → 执行对应检测项
- [ ] 环境不就绪时输出缺失项 + 修复建议（exit code 1, retryable）
- [ ] 用户修复后可重新检测
- [ ] Mobile 的 Maestro CLI 检测为非阻塞——未安装仍可生成 Maestro YAML

## User Stories

- **Story 3** (prd-user-stories.md): Surface 差异化 — 按 surface 类型检测环境

## Hard Rules

- 环境检测失败不自动修复，只输出诊断信息
- 检测项由 surface rule 文件指导，保持可扩展性
- Mobile 检测为 best-effort，不阻塞管线

## Implementation Notes

- 环境检测逻辑在 run-tests SKILL.md 中实现，通过读取 surface rule 文件获取检测项
- 检测结果可以作为 R2L（2.7）的前置条件——环境未就绪时跳过 R2L
