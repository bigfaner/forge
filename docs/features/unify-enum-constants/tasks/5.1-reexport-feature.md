# 5.1: Re-export typed constants from pkg/feature/

## Description

将 `pkg/feature/constants.go` 中的 Status 和 Priority 常量定义替换为从 `pkg/types/` 的重导出。此任务最后执行，因为 typed alias 会将常量类型从 untyped string 变为 `types.Status`/`types.Priority`，此时所有消费方已迁移完毕。

## Affected Files

- `forge-cli/pkg/feature/constants.go`

## Reference Files

- `forge-cli/pkg/feature/constants.go`: 移除原始 Status (7) 和 Priority (3) untyped 常量定义; 添加 type alias `type Status = types.Status`, `type Priority = types.Priority`; 添加 typed const re-export (source: design/tech-design.md#Interface-4)

## Acceptance Criteria

- [ ] `pkg/feature/constants.go` 中不再有原始的 Status/Priority 字符串常量定义
- [ ] 添加 `type Status = types.Status` 和 `type Priority = types.Priority` type alias
- [ ] 所有 7 个 Status 常量（`StatusPending` 等）重导出自 `types.StatusXxx`
- [ ] 所有 3 个 Priority 常量（`PriorityP0` 等）重导出自 `types.PriorityXxx`
- [ ] 路径常量（`FeaturesDir`、`PRDDirName` 等）保持不变
- [ ] `go build ./...` 全项目零错误
- [ ] `go test ./...` 全项目通过
- [ ] `feature.StatusPending == types.StatusPending` 为 true（type alias 保证）

## User Stories

- US3: Reliable Enum Refactoring — 重导出确保向后兼容
- US1: Type-Safe Status Constants — 统一类型来源

## Hard Rules

- 使用 type alias（`=`）而非 type definition，确保 `feature.Status` 与 `types.Status` 是同一类型
- 路径常量不动

## Implementation Notes

### Test Impact
- Affected test suite(s): `forge-cli/pkg/feature/` (constants tests if any)
- Expected fixture changes: none expected — type alias preserves value equality
- Risk level: low — single file change, all consumers already migrated

验证步骤：`forge grep -r "feature\.Status" forge-cli/` 应返回零结果（所有使用方已迁移为 `types.StatusXxx`）。
