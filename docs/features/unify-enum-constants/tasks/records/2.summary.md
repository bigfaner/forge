### 4. Conventions Established

- **Boundary pattern**: Structs that cross package boundaries (e.g., AddTaskOpts, AutoGenTaskDef) keep string fields; conversion happens at the point of use. This avoids forcing typed constant imports on callers.
- **String conversion pattern**: Use `string(types.PriorityP1)` when assigning typed constant to a string field, not `.String()` or `fmt.Sprintf`.
- **Wildcard convention**: `types.Status("*")` for match-all in transition rules — no dedicated StatusAny constant.
- **Zero behavior change**: All string values remain identical; only compile-time type safety improved.

### 5. Deviations from Design

- None. Implementation followed the tech-design Phase 3 plan (mapped to task IDs 2.1 and 2.2) exactly. The isTerminalStatus alignment with BIZ-task-lifecycle-001 (including skipped as terminal) was explicitly documented in the design as a known behavior change.
