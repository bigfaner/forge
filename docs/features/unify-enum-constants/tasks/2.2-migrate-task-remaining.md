# 2.2: Migrate pkg/task/ remaining files

## Description

替换 `pkg/task/` 中除 statemachine/state/types 之外的文件中的 Status、Priority、SurfaceType 魔法值，升级函数签名。

## Affected Files

- `forge-cli/pkg/task/add.go`
- `forge-cli/pkg/task/deps.go`
- `forge-cli/pkg/task/build.go`
- `forge-cli/pkg/task/autogen.go`
- `forge-cli/pkg/task/record.go`
- `forge-cli/pkg/task/index.go`
- `forge-cli/pkg/task/tasktemplate.go`
- `forge-cli/pkg/task/types.go` (NewTaskIndex StatusEnum/PriorityEnum)

## Reference Files

- `forge-cli/pkg/task/add.go`: `terminalStatuses` map: `map[string]bool` → `map[types.Status]bool` 使用 types.IsTerminalStatus; 默认状态赋值 "pending" → types.StatusPending; 优先级验证 "P0"/"P1"/"P2" → types.PriorityXxx (source: design/tech-design.md#Migration-Plan-Phase-3)
- `forge-cli/pkg/task/deps.go`: `satisfiedStatuses` map: `map[string]bool` → `map[types.Status]bool` (source: design/tech-design.md#Migration-Plan-Phase-3)
- `forge-cli/pkg/task/autogen.go`: `uiSurfaceTypes` map: `map[string]bool` → `map[types.SurfaceType]bool`; 19处 Priority 硬编码 "P1"/"P2" → types.PriorityXxx; 默认 Status → types.StatusPending (source: design/tech-design.md#Migration-Plan-Phase-3)
- `forge-cli/pkg/task/types.go`: `NewTaskIndex()` 中 StatusEnum: `[]string{...}` → `[]types.Status{...}`; PriorityEnum 同理; `TestTypeTitle()` 签名 + switch cases → types.SurfaceType (source: design/tech-design.md#Model-2)
- `forge-cli/pkg/task/tasktemplate.go`: `taskTemplateDefaults` map: 优先级 "P0" → types.PriorityP0 (source: design/tech-design.md#Migration-Plan-Phase-3)

## Acceptance Criteria

- [ ] `add.go` 中 `terminalStatuses` map 使用 `types.IsTerminalStatus()` 或 `map[types.Status]bool{}`
- [ ] `add.go` 中所有 Status/Priority 赋值和比较使用 typed constants
- [ ] `deps.go` 中 `satisfiedStatuses` map 使用 `types.StatusXxx`
- [ ] `build.go` 中默认状态赋值使用 `types.StatusPending`
- [ ] `autogen.go` 中 19 处 Priority 魔法值替换为 `types.PriorityP1`/`types.PriorityP2`
- [ ] `autogen.go` 中 `uiSurfaceTypes` 使用 `types.SurfaceType`
- [ ] `types.go` 中 `NewTaskIndex()` StatusEnum 为 `[]types.Status{}`，PriorityEnum 为 `[]types.Priority{}`
- [ ] `types.go` 中 `TestTypeTitle()` 参数和 switch cases 使用 `types.SurfaceType`
- [ ] `tasktemplate.go` 中优先级使用 `types.PriorityP0`
- [ ] `record.go`、`index.go` 中所有 Status 魔法值已替换
- [ ] `go build ./pkg/task/...` 零错误
- [ ] `go test ./pkg/task/...` 全部通过

## User Stories

- US1: Type-Safe Status Constants — add.go/deps.go 中的状态判断类型化
- US2: Centralized SurfaceType Constants — autogen.go uiSurfaceTypes, types.go TestTypeTitle
- US4: Validation Consolidation — NewTaskIndex StatusEnum/PriorityEnum 使用 typed slices

## Hard Rules

- `go build ./pkg/task/...` 和 `go test ./pkg/task/...` 必须通过
