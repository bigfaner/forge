# 2.1: Migrate pkg/task/ core types and statemachine

## Description

将 `pkg/task/` 中的核心类型和状态机从 `string` 升级为 typed constants。重点修改 `TransitionRule` 结构体、状态转移表、`matchRule`/`ValidateTransition` 函数签名、`Task` 结构体字段类型。

## Affected Files

- `forge-cli/pkg/task/statemachine.go`
- `forge-cli/pkg/task/state.go`
- `forge-cli/pkg/task/types.go` (struct fields only)

## Reference Files

- `forge-cli/pkg/task/statemachine.go`: TransitionRule struct From/To fields: `string` → `types.Status`; 整张 transitionTable (28 entries) 使用 types.StatusXxx; isTerminalStatus 改用 types.IsTerminalStatus; matchRule/ValidateTransition 签名升级 (source: design/tech-design.md#Model-1)
- `forge-cli/pkg/task/state.go`: switch/case 状态比较使用 types.StatusXxx; 函数签名 string → types.Status (source: design/tech-design.md#Interface-1)
- `forge-cli/pkg/task/types.go`: Task struct Status field: `string` → `types.Status`; Priority field: `string` → `types.Priority` (source: design/tech-design.md#Model-2)

## Acceptance Criteria

- [ ] `TransitionRule.From` 和 `TransitionRule.To` 类型为 `types.Status`
- [ ] `transitionTable` 中所有 28 个状态值使用 `types.StatusXxx` 常量（零字符串字面量）
- [ ] 通配符 `"*"` 保留为 `types.Status("*")` 或定义 `StatusAny` 常量，`matchRule` 比较逻辑兼容
- [ ] `isTerminalStatus` 函数改用 `types.IsTerminalStatus()`
- [ ] `ValidateTransition(current, target string, ...)` 签名变为 `ValidateTransition(current, target types.Status, ...)`
- [ ] `matchRule` 函数参数类型升级
- [ ] `Task` 结构体 `Status` 字段类型为 `types.Status`，`Priority` 字段类型为 `types.Priority`
- [ ] `TransitionError.From`/`To` 字段类型为 `types.Status`，`Error()` 输出格式不变
- [ ] `go build ./pkg/task/...` 编译通过（此时可能有其他文件报错，仅确保 statemachine.go + state.go + types.go 编译通过）
- [ ] `go test ./pkg/task/...` 运行 statemachine 相关测试验证状态转移正确

## User Stories

- US1: Type-Safe Status Constants — statemachine 是 Status 类型安全的核心验证点
- US3: Reliable Enum Refactoring — 编译器验证 TransitionRule 类型一致性

## Hard Rules

- `transitionTable` 的行为必须与原表完全一致（零行为变更）
- 通配符 `"*"` 必须继续工作

## Implementation Notes

### Test Impact
- Affected test suite(s): `forge-cli/pkg/task/` (existing statemachine tests)
- Expected fixture changes: test assertions comparing string status values may need type conversion
- Risk level: medium — 28 entries in transition table, high density of changes

通配符处理：建议使用 `types.Status("*")` 而非新增 `StatusAny` 常量，避免引入非标准状态值。`matchRule` 函数中 `r.From == "*"` 需改为 `r.From == types.Status("*")`。

IsTerminalStatus 行为变更：`skipped` 被纳入终态。需验证 `canAutoUnblock` 在 source task 为 skipped 时的行为是否符合 `BIZ-task-lifecycle-001`。
