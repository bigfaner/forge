# 4.1: Migrate internal/cmd/task/ Status and Priority constants

## Description

替换 `internal/cmd/task/` 中所有 Status 和 Priority 魔法值为 typed constants，升级函数签名。这是 Status 魔法值第二密集的区域（submit.go 17 处、validate_index.go 10 处）。

## Affected Files

- `forge-cli/internal/cmd/task/submit.go`
- `forge-cli/internal/cmd/task/claim.go`
- `forge-cli/internal/cmd/task/validate_index.go`
- `forge-cli/internal/cmd/task/add.go`
- `forge-cli/internal/cmd/task/reopen.go`
- `forge-cli/internal/cmd/task/transition.go`
- `forge-cli/internal/cmd/task/tree.go`
- `forge-cli/internal/cmd/task/migrate.go`

## Reference Files

- `forge-cli/internal/cmd/task/submit.go`: 17 处 Status 魔法值 ("completed", "blocked", "skipped", "pending"); 原 feature.StatusCompleted/feature.StatusSkipped 引用改为 types.StatusCompleted/types.StatusSkipped (source: design/tech-design.md#Migration-Plan-Phase-5)
- `forge-cli/internal/cmd/task/validate_index.go`: `validStatus` map (7 entries) → `types.AllStatuses()`; `validPriority` map (3 entries) → `types.AllPriorities()`; 额外 3 处内联比较 (source: design/tech-design.md#Migration-Plan-Phase-5)
- `forge-cli/internal/cmd/task/claim.go`: `priorityOrder` map: `map[string]int` → `map[types.Priority]int` 使用 typed constants; 7 处 Status 比较 (source: design/tech-design.md#Migration-Plan-Phase-5)
- `forge-cli/internal/cmd/task/migrate.go`: `feature.StatusInProgress` 引用改为 `types.StatusInProgress` (source: design/tech-design.md#Migration-Plan-Phase-5)

## Acceptance Criteria

- [ ] `submit.go` 中 17 处 Status 魔法值替换为 `types.StatusXxx`，原 `feature.StatusXxx` 引用改为 `types.StatusXxx`
- [ ] `validate_index.go` 中 `validStatus`/`validPriority` map 改用 `types.AllStatuses()`/`types.AllPriorities()`
- [ ] `claim.go` 中 `priorityOrder` map 使用 `map[types.Priority]int{types.PriorityP0: 0, ...}`
- [ ] `add.go`、`reopen.go`、`transition.go`、`tree.go` 中所有 Status/Priority 魔法值替换
- [ ] `migrate.go` 中 `feature.StatusInProgress` 改为 `types.StatusInProgress`
- [ ] CLI flag 默认值边界转换：`"P1"` flag default 转为 `types.Priority("P1")` 或直接用 `string(types.PriorityP1)`
- [ ] `go build ./internal/cmd/task/...` 零错误
- [ ] `go test ./internal/cmd/task/...` 全部通过

## User Stories

- US1: Type-Safe Status Constants — submit.go 和 validate_index.go 是 Status 类型安全的最大受益者
- US4: Validation Consolidation — validStatus/validPriority 改用 AllStatuses/AllPriorities

## Hard Rules

- CLI 行为零变更——所有 flag 输入输出保持兼容

## Implementation Notes

### Test Impact
- Affected test suite(s): `forge-cli/internal/cmd/task/` (submit_test, claim_test, validate_index_test, add_test, etc.)
- Expected fixture changes: test string literals → types.StatusXxx
- Risk level: high — 8 files, ~50+ replacements

边界转换：cobra flag 返回 string，需要在赋值点做 `types.Status(flagValue)` 或 `types.Priority(flagValue)` 转换。
