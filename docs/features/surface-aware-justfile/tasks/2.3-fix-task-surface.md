# 2.3: quality-gate fix-task surface 推断

## Description

更新 quality-gate 相关 skill 文档，使 fix-task 流程从失败文件路径调用 `forge surfaces --json <path>` 推断 surface-key/type，替代原有的 scope 推断逻辑。

## Reference Files

`design/tech-design.md#Phase-Component-Map` — Phase 2 quality-gate 变更
`design/tech-design.md#PRD-Coverage-Map` — Story4: fix-task 推断

## Acceptance Criteria

- Given quality-gate 检测到 `src/components/App.tsx` 测试失败并创建 fix-task
- When fix-task 创建流程执行
- Then 调用 `forge surfaces --json src/components/App.tsx` 获取 surface-key/type
- And fix-task frontmatter 包含推断的 surface-key 和 surface-type
- And CLI 调用失败时 fallback 为空值，不阻塞 fix-task 创建

## User Stories

- Story 4 (Task 数据模型扩展): fix-task 从失败文件路径推断 surface-key/type

## Implementation Notes

推断逻辑在 skill 层实现（LLM agent 调用 Bash 执行 `forge surfaces --json`）。不涉及 quality_gate.go Go 代码变更（PRD Out of Scope）。
