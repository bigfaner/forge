# 3.4: run-tests surface 规则文件（5 个）

## Description

创建 run-tests 的 5 个 surface 执行策略规则文件。每个文件定义该 surface 类型的编排序列、退出码语义和执行约束。

## Reference Files

`design/tech-design.md#Interface-2` — Surface 规则文件完整格式（编排序列表）
`design/tech-design.md#PRD-Coverage-Map` — Story2 编排序列

## Affected Files

### Create
| File | Description |
|------|-------------|
| `skills/run-tests/rules/surfaces/web.md` | web 编排：dev→probe→test→teardown |
| `skills/run-tests/rules/surfaces/api.md` | api 编排：dev→probe→test→teardown |
| `skills/run-tests/rules/surfaces/cli.md` | cli 编排：build→test |
| `skills/run-tests/rules/surfaces/tui.md` | tui 编排：build→test |
| `skills/run-tests/rules/surfaces/mobile.md` | mobile 编排：test-setup→dev→test→teardown |

## Acceptance Criteria

- web/api 规则文件包含 4 步编排序列及每步的 exit code 语义（0/1/2）
- cli/tui 规则文件包含 2 步编排（无 dev/probe）
- mobile 规则文件包含 test-setup 前置步骤
- 每个文件定义 probe 失败 HARD-GATE 约束
- 规则文件可被 run-tests SKILL.md 直接加载消费

## Implementation Notes

格式与 init-justfile 规则文件保持一致（共享 Interface 2 定义的编排序列表结构）。run-tests 主要消费编排序列部分，配方调用契约和 journey 过滤为 init-justfile 使用。
