# 3.2: init-justfile surface 规则文件（5 个）

## Description

创建 init-justfile 的 5 个 surface 规则文件：web.md、api.md、cli.md、tui.md、mobile.md。每个文件定义该 surface 类型的编排序列、配方调用契约和 journey 过滤策略。

## Reference Files

`design/tech-design.md#Interface-2` — 完整 web.md 示例（编排序列表、配方调用契约表、journey 过滤策略表、实现约束）
`design/tech-design.md#Interface-2` — 其余 surface 类型差异点

## Affected Files

### Create
| File | Description |
|------|-------------|
| `skills/init-justfile/rules/surfaces/web.md` | web surface 规则（dev→probe→test→teardown） |
| `skills/init-justfile/rules/surfaces/api.md` | api surface 规则（dev→probe→test→teardown，probe 检查 /healthz） |
| `skills/init-justfile/rules/surfaces/cli.md` | cli surface 规则（build→test，无 dev/probe） |
| `skills/init-justfile/rules/surfaces/tui.md` | tui surface 规则（build→test，无 dev/probe） |
| `skills/init-justfile/rules/surfaces/mobile.md` | mobile surface 规则（test-setup→dev→test→teardown） |

## Acceptance Criteria

- web.md 包含完整的编排序列表（4 步）、配方调用契约表（5 个配方含聚合配方）、journey 过滤策略表
- api.md 与 web.md 结构相同，probe 目标为 /healthz，支持 @api journey
- cli.md 和 tui.md 无 dev/probe 步骤，无聚合配方
- mobile.md 包含 test-setup 步骤（模拟器准备），支持 @mobile journey
- 每个文件的格式严格遵循 Interface 2 定义的 markdown 结构

## Implementation Notes

以 tech-design Interface 2 的 web.md 完整示例为模板，按各 surface 类型差异点调整。web/api 使用 4 步编排，cli/tui 使用 2 步编排，mobile 使用 test-setup 前置步骤。
