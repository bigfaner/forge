# 1.4: resolveScope 删除 & prompt 变量源切换

## Description

删除 `prompt.go` 中的 `resolveScope()` 函数，将 `{{SCOPE}}` 模板变量的值来源从 resolveScope 计算切换为直接读取 `task.SurfaceKey`。模板变量名暂时保持 `{{SCOPE}}` 不变（Phase 3 任务 3.5 统一重命名为 `{{SURFACE_KEY}}`）。

## Reference Files

`design/tech-design.md#Phase-Component-Map` — Phase 1 prompt.go 变更说明
`design/tech-design.md#PRD-Coverage-Map` — Story3: resolveScope 重写 → 删除

## Acceptance Criteria

- Given `resolveScope()` 函数已删除
- When `renderTemplate()` 渲染包含 `{{SCOPE}}` 的模板
- Then `{{SCOPE}}` 被替换为 `task.SurfaceKey` 值（空字符串表示跨 surface）
- And `extractTestTypeArg()` 继续工作，从 `task.SurfaceType` 生成 `--type <type>` 参数
- And `go test ./pkg/prompt/...` 全部通过
- And 所有现有 prompt 模板输出与变更前一致（仅值来源变化，变量名不变）

## User Stories

- Story 3 (surface-key 迁移): resolveScope 删除，直接读 SurfaceKey

## Implementation Notes

当前 `renderTemplate()` 中 `{{SCOPE}}` 替换逻辑（line ~137）：`strings.ReplaceAll(result, "{{SCOPE}}", scope)`，其中 scope 由 resolveScope 计算。改为直接使用 task.SurfaceKey 值。extractTestTypeArg() 改为从 task.SurfaceType 取值。
