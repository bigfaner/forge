# 3.1: init-justfile SKILL.md surface 感知重写

## Description

重写 init-justfile SKILL.md，新增 surface 检测步骤和 surface 感知配方生成流程。按 surface-type 加载对应规则文件生成差异化的 dev/test/probe/teardown 配方。

## Reference Files

`design/tech-design.md#Interface-2` — Surface 规则文件完整格式（web.md 示例）
`design/tech-design.md#PRD-Coverage-Map` — Story1 配方生成、双平台变体、CLI/TUI 不生成 run

## Acceptance Criteria

- Given 项目 config.yaml 定义 `surfaces: {admin-panel: web}`
- When 运行 init-justfile
- Then 生成的 justfile 包含 dev、probe、test、teardown 配方
- And 每个配方支持 `[linux]`/`[windows]` 双平台变体
- And CLI/TUI surface 不生成 dev/probe 配方
- And 混合项目（web+api）生成带 surface-key 前缀的独立配方 + 聚合配方
- And `# user-customized` 标记的配方不被覆盖
- And 无 surface 配置的项目输出与当前完全一致（零回归）
- And `just --dry-run` 验证生成的 justfile 语法正确

## User Stories

- Story 1 (surface 感知配方生成): 全部 AC

## Hard Rules

- surface-key 命名约束 `[a-zA-Z0-9_-]` 由配方生成时强制执行
- just >= 1.4.0 版本检查作为首步

## Implementation Notes

生成流程参考 PRD Flow Description：检测语言 → 加载语言模板 → 检测 surface → 加载 surface 规则 → 仲裁（Surface 规则覆盖语言模板的编排级配方）→ 保护 user-customized → 跨平台变体验证 → 组装。混合项目配方前缀格式：`dev-<surface-key>`。
