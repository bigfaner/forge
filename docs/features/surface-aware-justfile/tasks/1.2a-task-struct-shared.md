# 1.2a: Task struct 字段迁移 — 共享构件更新

## Description

将 Task/AutoGenTaskDef/FrontmatterData/AddTaskOpts/TaskState 五个结构体的 `Scope` 字段删除，新增 `SurfaceKey` + `SurfaceType` 字段。修改所有 Go 消费方以通过编译——此任务仅处理字段声明变更和编译适配，不实现传播逻辑。

## Reference Files

`design/tech-design.md#Model-1` — Task struct SurfaceKey/SurfaceType 字段定义、JSON tag
`design/tech-design.md#Model-2` — AutoGenTaskDef 字段变更
`design/tech-design.md#Model-3` — FrontmatterData 字段变更
`design/tech-design.md#Model-4` — AddTaskOpts 字段变更
`design/tech-design.md#Model-5` — TaskState 字段变更
`design/tech-design.md#Migration-Notes` — Scope 字段删除的设计决策和 PRD AC 更新

## Acceptance Criteria

- Given 五个结构体的 Scope 字段已删除
- When 编译整个项目
- Then `go build ./...` 成功，无编译错误
- And `Task` struct 包含 `SurfaceKey string` (JSON: `"surface-key,omitempty"`) 和 `SurfaceType string` (JSON: `"surface-type,omitempty"`)
- And `AutoGenTaskDef` 删除 Scope 和 TestType，新增 SurfaceKey + SurfaceType（含 yaml/json tag）
- And `FrontmatterData` 删除 Scope，新增 SurfaceKey + SurfaceType
- And `AddTaskOpts` 新增 SurfaceKey + SurfaceType
- And `TaskState` 删除 Scope，新增 SurfaceKey + SurfaceType
- And 所有消费这些结构体的代码通过编译（使用临时空值或占位符）

## User Stories

- Story 3 (surface-key 迁移): 核心数据模型变更
- Story 4 (Task 数据模型扩展): 双字段新增

## Hard Rules

- 不保留 Scope 字段的任何兼容层或别名
- 不实现传播/解析逻辑（留给 1.2b）

## Implementation Notes

这是 existing-code-split 的 Sub-task A。受影响文件：`task/types.go`、`task/autogen.go`、`task/frontmatter.go`、`task/add.go`、`prompt.go`。消费方适配可能涉及 `build.go`（frontmatter 解析）、`quality_gate.go` 等文件。JSON tag 遵循 `"surface-key,omitempty"` / `"surface-type,omitempty"` 格式。
