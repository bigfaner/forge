## Conventions Established

1. **Surface 推断统一通过 forgeconfig Go 库** — 直接调用 ReadSurfaces()+MatchSurface()，不通过 CLI 子进程（2.3 确立，2.1 使用 CLI 是因为 skill 是 markdown 无法调用 Go 库）
2. **Surface 推断失败不阻塞** — 推断失败回退为空值，绝不阻塞任务创建
3. **模板用静态空值 + 后注入** — 模板中 surface-key/surface-type 使用空占位，运行时通过 injectSurfaceFrontmatter() 覆盖
4. **Skill 文档中 surface 解析使用 CLI** — SKILL.md 中通过 forge surfaces --json <path> 调用，因为 skill 是 agent 执行的 markdown 而非 Go 代码

## Deviations from Design

- **[2.3] Go 库调用 vs CLI 子进程**：设计文档中 quality-gate 相关逻辑描述为调用 forge surfaces CLI，实际实现使用 Go 库 forgeconfig.ReadSurfaces()+MatchSurface() 直接调用。偏离原因：quality_gate.go 本身是 Go 代码，库调用比子进程更可靠高效，且 AC 明确要求 forge surfaces --json 调用推断 surface，Go 库调用满足同等语义
- **[2.3] Implementation Notes 与 AC 冲突**：Implementation Notes 称 'no quality_gate.go Go code changes'，但 AC 和 PRD Coverage Map 明确要求 addFixTask() 调用 forge surfaces 推断。遵循 AC 作为更具体的需求，实际修改了 quality_gate.go
- **[2.2] 任务重分类为验证**：设计预期 quick-tasks 需独立修改，但 2.1 已将 breakdown-tasks 和 quick-tasks 一并更新，使 2.2 变为纯验证任务
