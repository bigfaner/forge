# 3.5: Prompt 模板变量迁移

## Description

将所有 prompt 模板和 skill 模板中的 `{{SCOPE}}` 全局替换为 `{{SURFACE_KEY}}`，`{{TEST_TYPE}}` 替换为 `{{TEST_TYPE_ARG}}`。同步更新 prompt.go renderTemplate() 中的替换逻辑。

## Reference Files

`design/tech-design.md#Phase-Component-Map` — Phase 3 prompt 模板清单（18 prompt/data + 3 skill 模板）
`design/tech-design.md#Phase-Component-Map` — Prompt 模板完整清单表格（含变量使用列）

## Acceptance Criteria

- Given 所有模板文件已更新
- When `renderTemplate()` 渲染任意模板
- Then `{{SURFACE_KEY}}` 被替换为 `task.SurfaceKey` 值
- And `{{TEST_TYPE_ARG}}` 被替换为 `" --type <type>"` 或空字符串
- And 不存在任何模板文件仍包含 `{{SCOPE}}` 或 `{{TEST_TYPE}}`
- And `go test ./pkg/prompt/...` 全部通过
- And 生成的 prompt 内容与迁移前语义一致

## User Stories

- Story 3 (surface-key 迁移): 变量名统一为 SURFACE_KEY

## Implementation Notes

受影响文件清单见 tech-design Phase Component Map 的 Prompt 模板完整清单表格。18 个 `prompt/data/*.md` 全部使用 `{{SCOPE}}`，其中仅 `test-gen-scripts.md` 同时使用 `{{TEST_TYPE_ARG}}`。3 个 skill 模板使用 `{{SCOPE}}`。prompt.go 中 `renderTemplate()` 的替换行从 `{{SCOPE}}` 改为 `{{SURFACE_KEY}}`。
