# 2.1: breakdown-tasks surface-key/type 填充

## Description

更新 breakdown-tasks SKILL.md，新增任务生成时调用 `forge surfaces --json <path>` 获取 SurfaceKey + SurfaceType 并写入任务 frontmatter 的步骤。新增规则文件指导 agent 将 scope 枚举映射替换为 CLI 动态查询。

## Reference Files

`design/tech-design.md#Phase-Component-Map` — Phase 2 breakdown-tasks 变更
`design/tech-design.md#Interface-1b` — forge surfaces --json 输出格式
`design/tech-design.md#Cross-Layer-Data-Map` — surface 信息从 CLI 到 frontmatter 的传递

## Acceptance Criteria

- Given breakdown-tasks 生成涉及 `src/components/App.tsx` 的任务
- When 任务 frontmatter 生成完成
- Then frontmatter 包含 `surface-key: admin-panel` 和 `surface-type: web`
- And 无 surfaces 配置的项目不填充 surface-key/type（空值）
- And `forge surfaces --json` 调用失败时 skill 输出错误消息含恢复提示

## User Stories

- Story 3 (surface-key 迁移): 任务生成时填充 surface-key
- Story 4 (Task 数据模型扩展): breakdown-tasks 自动填充双字段

## Implementation Notes

SKILL.md 新增步骤：遍历任务涉及文件 → 调用 `forge surfaces --json <file-path>` → 解析 JSON → 写入 frontmatter surface-key + surface-type。新规则文件替代现有 `rules/scope-assignment.md` 中的硬编码路径分类逻辑。
