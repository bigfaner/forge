# 1.3: forge task migrate + CheckLegacyScope

## Description

新增 `task/migrate.go` 中的 `CheckLegacyScope()` 共享函数和 `forge task migrate` 子命令。CheckLegacyScope 供所有 task 读取路径调用，检测旧 scope 字段并阻塞；forge task migrate 自动将旧 scope 映射为 surface-key + surface-type。

## Reference Files

`design/tech-design.md#Migration-Notes` — 迁移验证步骤、CheckLegacyScope 覆盖范围表、CI 升级影响与缓解
`design/tech-design.md#Error-Types-&-Codes` — 阻塞错误 exit 2 场景

## Acceptance Criteria

- Given index.json 包含任务含 `scope: frontend` 但无 `surface-key`
- When `CheckLegacyScope()` 被调用
- Then 返回错误，stderr 输出 `migration required: found N tasks with legacy 'scope' field but no 'surface-key' — run 'forge task migrate' or 'forge breakdown-tasks' to regenerate tasks`
- And exit code 2 (blocking)
- Given 运行 `forge task migrate`
- When index.json 包含含 scope 但无 surface-key 的任务
- Then 通过 `forge surfaces --json <path>` 将 scope 映射为 surface-key + surface-type
- And 就地更新 index.json 和对应 frontmatter 文件
- And CheckLegacyScope 覆盖 BuildIndex()、ListTasks()、ShowTask()、AddTask() 四个读取路径
- And `go test ./pkg/task/... ./internal/cmd/...` 全部通过

## User Stories

- Story 4 (Task 数据模型扩展): 旧任务 scope 迁移机制

## Hard Rules

- CheckLegacyScope 必须是共享函数，不能仅在 build.go 中内联
- 错误消息必须包含恢复提示（BIZ-error-reporting-002）

## Implementation Notes

CheckLegacyScope 函数签名：`func CheckLegacyScope(tasks []Task) error`，放在 `task` 包中。调用点参考 tech-design Migration Notes 的读取路径覆盖表。forge task migrate 子命令注册在 `internal/cmd/task.go` 或新建文件。
