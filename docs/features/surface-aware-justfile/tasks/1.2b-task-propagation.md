# 1.2b: Task struct 字段迁移 — 传播与解析

## Description

实现 SurfaceKey/SurfaceType 在 Go 层的传播链：autogen.go 的任务定义→Task 传播、frontmatter.go 的 frontmatter 解析/序列化、add.go 的源任务字段复制。

## Reference Files

`design/tech-design.md#Model-2` — AutoGenTaskDef SurfaceKey/SurfaceType 传播链
`design/tech-design.md#Model-3` — FrontmatterData 解析逻辑
`design/tech-design.md#Model-4` — AddTaskOpts 从源任务继承
`design/tech-design.md#Cross-Layer-Data-Map` — surface-key/type 跨层传递

## Acceptance Criteria

- Given AutoGenTaskDef 包含 SurfaceKey="admin-panel", SurfaceType="web"
- When 通过 TaskFromFile() 映射为 Task
- Then Task.SurfaceKey="admin-panel", Task.SurfaceType="web"
- And frontmatter 解析能读取 `surface-key: admin-panel` 和 `surface-type: web`
- And frontmatter 序列化输出包含 `surface-key:` 和 `surface-type:` 字段
- And AddTaskOpts 从源任务正确复制 SurfaceKey 和 SurfaceType
- And `go test ./pkg/task/...` 全部通过
- And 旧 frontmatter 不含 surface-key 时不报错（空值正常处理）

## User Stories

- Story 3 (surface-key 迁移): 传播链实现
- Story 4 (Task 数据模型扩展): frontmatter 序列化/反序列化
- Story 4: forge task add 继承 surface-key/surface-type

## Implementation Notes

autogen.go 当前在 `TaskFromFile()` 中映射 `Scope: d.Scope`（line ~841），需改为 `SurfaceKey: d.SurfaceKey, SurfaceType: d.SurfaceType`。frontmatter.go 的 `ParseFrontmatter` 需新增 surface-key/surface-type YAML 字段解析。add.go 的 `AddTask` 需从 source task 复制双字段。
