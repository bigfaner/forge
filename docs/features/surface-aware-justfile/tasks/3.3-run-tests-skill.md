# 3.3: run-tests SKILL.md 调度器模式重写

## Description

重写 run-tests SKILL.md 为调度器模式：检测 surface-type → 加载对应执行策略规则 → 按序列执行 just 配方。移除 test.execution 读取逻辑。

## Reference Files

`design/tech-design.md#PRD-Coverage-Map` — Story2 调度器模式编排、probe 失败 exit code
`design/tech-design.md#Interface-2` — Surface 规则文件格式（run-tests 消费编排序列）

## Acceptance Criteria

- Given 任务 frontmatter 包含 `surface-type: web`
- When run-tests 执行
- Then 按序列执行 just dev → probe → test → teardown
- And probe 失败（exit 1 retryable）后执行 teardown + exit 1
- And probe 失败（exit 2 blocking）后执行 teardown + exit 2
- And probe 失败后禁止在同一编排周期内重试 probe 或重启 dev（HARD-GATE）
- And surface 信息不可用时（两个来源均失败）stderr 输出错误含恢复提示 + exit 2
- And CLI/TUI surface 执行 build → dev → test（无 probe/teardown）

## User Stories

- Story 2 (自动化测试编排): 全部 AC

## Hard Rules

- probe 失败 HARD-GATE：不可违反
- exit code 语义严格遵循 BIZ-error-reporting-001

## Implementation Notes

参考 PRD Flow Description 的 run-tests 编排流程：获取 surface 信息（优先 frontmatter → forge surfaces CLI）→ 加载规则 → 编排执行 → 每步检查退出码。移除 config.yaml 中 test.execution 的读取逻辑。Probe 重试参数：max 3 次，间隔 30 秒。
