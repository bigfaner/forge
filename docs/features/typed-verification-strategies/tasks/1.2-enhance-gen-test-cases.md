# 1.2: Enhance gen-test-cases SKILL.md for typed verification

## Description

Add interface-type awareness to the gen-test-cases skill: read profile verification strategies, generate type-specific test cases with Level/Interface fields, validate strategy format, and handle errors gracefully.

This is the core pipeline change — modifies SKILL.md (Steps 2.7, 3, 4), the test-cases.md template, and adds error handling.

## User Stories

- **Story 1**: As a Forge user, I want gen-test-cases to generate type-specific verification conditions
  - Given a go-test profile with verification-strategies.md
  - When gen-test-cases executes
  - Then TUI TCs include golden file assertions + dimension checks + >=2 boundary scenarios, API TCs include contract verification + error paths + >=2 boundary values, CLI TCs include output golden file + exit codes + parameter combinations

- **Story 2**: As a Forge user, I want auto Level tagging
  - Given gen-test-cases has read the strategy file
  - When generating TCs
  - Then TUI/web-ui/mobile-ui TCs have Level=e2e, API/CLI TCs have Level=integration, Level field coverage >= 95%

- **Story 3**: As a Profile author, I want format validation
  - Given a strategy file with incomplete sections
  - When gen-test-cases executes
  - Then execution aborts with validation error listing missing subsections

- **Story 4**: As a Forge user, I want graceful degradation
  - Given a profile without verification-strategies.md
  - When gen-test-cases executes
  - Then warning output, generic TCs without Level field, execution continues

- **Story 5**: As a Profile author, I want clear key mismatch errors
  - Given strategy file with tui+api but manifest declares tui+cli
  - When gen-test-cases executes
  - Then abort with "Strategy/manifest mismatch. Missing in strategy: cli. Extra in strategy: api."

- **Story 7**: As a Forge user, I want Level + Interface fields in test-cases.md
  - Given a go-test profile with valid strategy file
  - When gen-test-cases completes
  - Then each TC has Level and Interface fields matching the capability mapping
