## Implementation Notes

Read each profile's `manifest.yaml` first to determine capabilities, then read `generate.md` to understand the framework's testing patterns. Tailor verification dimensions and boundary scenarios to the framework:

- **web-playwright**: DOM interaction, visual regression, responsive layout, accessibility
- **maestro**: device rendering, touch interaction, screen orientation, platform differences
- **pytest**: fixture-based contract testing, parametrize boundary values, real dependency testing
- **rust-test**: ownership/borrowing edge cases, async runtime, error propagation
- **java-junit**: Spring context testing, JUnit lifecycle, parameterized tests

The go-test file serves as the structural reference (section format, item counts) but the content must be framework-specific.
