# 2.2: Update eval-test-cases rubric with Typed Verification Completeness dimension

## Description

Add a new "Typed Verification Completeness" scoring dimension (150 pts) to the eval-test-cases rubric. Redistribute points from existing dimensions to maintain 1000-point total. Update the report template to include the new dimension in the scorecard.

## Reference Files

- `plugins/forge/skills/eval-test-cases/templates/rubric.md` — current rubric (1000 pts, 5 dimensions)
- `plugins/forge/skills/eval-test-cases/templates/report.md` — report template with scorecard
- `docs/features/typed-verification-strategies/design/tech-design.md` — PRD Coverage Map S8 entry (scoring formula, 3 check items, fallback rule)
- `docs/features/typed-verification-strategies/prd/prd-spec.md` — PRD §Related Changes item 4
- `docs/features/typed-verification-strategies/prd/prd-user-stories.md` — Story 8

## Acceptance Criteria

- [ ] New dimension "Typed Verification Completeness" added to rubric with 150 pts total
- [ ] Point redistribution: PRD Traceability 225, Step Actionability 225, Interface Accuracy 180, Completeness 180, Typed Verification 150, Structure & ID Integrity 40 (total = 1000)
- [ ] Dimension has 3 check items matching design S8 specification:
  1. Dimension coverage: all strategy-defined verification dimensions present in TCs (>=3/3 per capability) — 50 pts
  2. Boundary scenario coverage: TCs include at least 2 scenarios from strategy definition — 50 pts
  3. Test data compliance: test data satisfies strategy's data requirements — 50 pts
- [ ] Scoring formula documented: `score = (covered_items / total_items) * weight`
- [ ] Fallback rule: when no strategy file exists, dimension score rate >= 0.8 (no penalty for missing strategy)
- [ ] Profile-adaptive: dimension only active when profile has verification-strategies.md
- [ ] Report template scorecard updated with new dimension row
- [ ] Required Sections checklist updated to include "Level and Interface fields in TC entries" (when strategy is active)

## User Stories

- **Story 8**: As a Forge user, I want eval-test-cases to check typed verification completeness
  - Given test-cases.md has TUI TCs with Interface=TUI, Level=e2e and profile strategy defines golden file + dimension checks + CJK boundaries
  - When eval-test-cases scores
  - Then report includes "Typed Verification Completeness" dimension checking (1) dimension coverage >=3/3, (2) boundary scenarios >=2, (3) test data compliance; score = covered/total * weight; no strategy → rate >= 0.8

## Implementation Notes

The current rubric has 5 dimensions totaling 1000 pts. After this change it will have 6 dimensions totaling 1000 pts. The redistribution reduces existing dimensions proportionally:

| Dimension | Current | New |
|-----------|---------|-----|
| PRD Traceability | 250 | 225 |
| Step Actionability | 250 | 225 |
| Interface Accuracy | 200 | 180 |
| Completeness | 200 | 180 |
| **Typed Verification Completeness** | — | **150** |
| Structure & ID Integrity | 100 | 40 |

The new dimension is profile-aware: read the active profile's verification-strategies.md to determine expected verification dimensions and boundary scenarios per capability, then check whether TCs cover them.
