# 1: Update brainstorm intent mapping to 6-value enum

## Description

Remove the heuristic logic that splits `coding.fix` into new-feature or refactor. Update AskUserQuestion to offer all 6 intent options. Update the proposal template's intent valid values comment.

## Implementation Notes
- The AskUserQuestion expansion from 3 to 6 options is a user-visible interaction change but non-breaking (old 3 values remain available)
- Per proposal Architecture Decision: `enhancement` maps to `coding.enhancement` (not `coding.feature`), restoring the semantic distinction between "adds new runtime behavior" vs "improves existing behavior"
- Per proposal: `doc.consolidate` and `doc.drift` are low-frequency internal task types auto-generated by skills, unified under `doc` umbrella — no separate intent needed
