# 4: Test record template (record-test.md)

## Description

Create `record-test.md` template for test-category tasks (`test.gen-cases`, `test.eval-cases`, `test.gen-scripts`, `test.run`, `test.gen-and-run`, `test.graduate`, `test.verify-regression`). Includes test-pipeline-specific metrics: Cases Generated/Evaluated, Scripts Created.

## Reference Files
- `docs/proposals/typed-task-records/proposal.md` — Source proposal
- `forge-cli/pkg/task/data/record-coding.md` — Reference template structure (from task 2)

## Acceptance Criteria
- [ ] `record-test.md` template file created in `forge-cli/pkg/task/data/`
- [ ] Template renders "Cases Generated" section using `.CasesGenerated` field
- [ ] Template renders "Cases Evaluated" section using `.CasesEvaluated` field
- [ ] Template renders "Scripts Created" section using `.ScriptsCreated` list
- [ ] Template renders "Test Results" section using `.TestResults` free-text field
- [ ] Template includes shared sections: Summary, Changes, Key Decisions, Acceptance Criteria, Notes
- [ ] `fillRecordTemplate()` routes test-category types to this template
- [ ] Unit tests verify test template output

## Hard Rules
- Test tasks are system-generated (auto-gen). Their records should be concise — focus on pipeline metrics, not verbose narratives.
- The `TestResults` field is free-text — the template just renders it as-is.

## Implementation Notes
- Test tasks are auto-generated by `forge task index` and typically run without human intervention. The record should capture what was generated/run, not subjective analysis.
