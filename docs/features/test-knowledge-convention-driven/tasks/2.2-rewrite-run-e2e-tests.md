# 2.2: Rewrite run-e2e-tests skill

## Description
Rewrite the run-e2e-tests SKILL.md to execute tests via `just e2e-test` and parse results using Convention-declared Result Format. Remove all `forge test detect` and `forge test get run` calls. Remove hardcoded framework-specific parsing logic.

## Reference Files
- `plugins/forge/skills/run-e2e-tests/SKILL.md` — current skill (to rewrite)
- `docs/features/test-knowledge-convention-driven/design/tech-design.md` — I-6 run-e2e-tests interface
- `docs/features/test-knowledge-convention-driven/prd/prd-spec.md` — FS-2, FS-4

## Acceptance Criteria
- SKILL.md contains zero references to `forge test detect`, `forge test get run`
- Step 0: Load Convention Result Format section (format-type + output-flags)
- Step 1: Execute `just e2e-test`
- Step 2: Parse results based on Convention-declared format type (json-stream, json-report, text-verbose)
- Fallback: no Convention → generic text-based parsing
- No hardcoded framework-specific parsing logic in SKILL.md

## User Stories
- Story 1 (non-default framework): Result Format drives parsing strategy

## Hard Rules
- Parsing logic must be driven by Convention Result Format section, not framework name

## Implementation Notes
- Three format types from Convention:
  - json-stream: Go `go test -json` — line-by-line JSON events
  - json-report: Python `pytest --json-report`, JS `playwright --reporter=json` — single JSON report
  - text-verbose: generic `-v` output — text parsing
- Skill prompt describes parsing strategy per format-type, not per framework
