# 1.1: Create pkg/forgeconfig package

## Description
Extract retained config types from `pkg/profile/config.go` into a new `pkg/forgeconfig/` package. This is an additive change — the new package coexists with `pkg/profile/` temporarily. Consumers will switch in subsequent tasks.

Preserve the `raw` map field in `AutoConfig` that tracks which YAML keys were explicitly set vs absent (needed to distinguish "explicitly set to false" from "not in YAML").

## Reference Files
- `forge-cli/pkg/profile/config.go` — source of types to extract
- `forge-cli/pkg/profile/config_test.go` — tests to transfer
- `docs/features/test-knowledge-convention-driven/design/tech-design.md` — I-1 interface definition

## Acceptance Criteria
- `pkg/forgeconfig/` package created with: `Config`, `AutoConfig` (including `Validation ModeToggle` and `raw` map), `ModeToggle`, `WorktreeConfig`
- Functions: `ReadConfig`, `ReadAutoConfig` (returns value type), `GetConfigValue`, `WriteConfig`
- `ReadConfig` parses only retained fields (`auto`, `worktree`); ignores unknown fields silently (yaml.Unmarshal default)
- Removed fields vs profile.ForgeConfig: `ProjectType`, `Languages`, `Interfaces`, `TestFramework`, `TestCommand`
- Existing profile tests migrated and passing
- `go build ./...` succeeds (profile package still exists, no consumers changed yet)

## User Stories
- Story 4 (backward compatibility): silent ignore of old config fields

## Hard Rules
- Preserve the `raw` map default-tracking mechanism — do not rewrite from scratch
- `ReadAutoConfig` must return value type `(AutoConfig, error)`, not pointer

## Implementation Notes
- Extract from `pkg/profile/config.go`, then simplify:
  - Remove ProjectType, Languages, Interfaces, TestFramework, TestCommand fields
  - Remove ReadLanguages, ReadInterfaces, WriteLanguages functions (not needed)
  - Keep ReadConfig, ReadAutoConfig, GetConfigValue, WriteConfig with simplified logic
  - Keep Validation ModeToggle field (used by pkg/task/testgen.go to gate validate-code/validate-ux tasks)
- Transfer relevant tests from `pkg/profile/config_test.go` and `pkg/profile/autoconfig_test.go`
