## 4. Conventions Established

- **Convention-first skill architecture**: All three test-related skills now follow a shared pattern: Step 0 loads Convention files from docs/conventions/ filtered by domains frontmatter. Framework/language knowledge is entirely delegated to Convention content, not hardcoded in skill prompts.
- **Unified compile gate via just**: gen-test-scripts uses `just e2e-compile` as the single compile verification mechanism, replacing framework-specific commands (go build, pytest --collect-only, tsc).
- **Format-type driven parsing**: run-e2e-tests parses results based on Convention-declared format-type rather than detecting framework name. Three supported formats: json-stream, json-report, text-verbose.
- **Cold start with LLM defaults**: When no Convention files exist, skills proceed with LLM-generated defaults based on file signal detection, outputting a hint to run test-guide to create Convention files.
- **Convention validation with graceful degradation**: Missing Convention sections produce warnings and fall back to LLM defaults. Malformed Convention files are skipped with warnings. Skills never hard-fail on Convention issues.
- **Template path using CLAUDE_SKILL_DIR**: Template references use ${CLAUDE_SKILL_DIR}/templates/ for portability across distributed environments, per forge-distribution.md.
- **No framework-specific templates or branches**: Hard Rule enforced across all three skills — zero hardcoded framework branch logic. All framework differences are described by Convention content.
