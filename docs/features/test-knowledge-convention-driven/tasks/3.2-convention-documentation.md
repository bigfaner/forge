# 3.2: Define Convention file structure documentation

## Description
Document the Convention file fixed structure, section schema, and validation rules for users. This serves as the reference that `/forge:test-guide` generates and gen-test-scripts consumes.

## Reference Files
- `docs/features/test-knowledge-convention-driven/design/tech-design.md` — Data Models section
- `docs/features/test-knowledge-convention-driven/prd/prd-spec.md` — FS-1

## Acceptance Criteria
- Convention file structure documented with: required sections, field-level schema, validation rules
- Examples for Go testing, Python pytest, JavaScript vitest
- Documented merge semantics (section-level, last-loaded wins)
- Documented growth path: minimal set → extended sections

## User Stories
- Story 2 (Convention bootstrap): users understand what test-guide generates
- Story 5 (multi-framework): users understand how multiple Convention files interact
