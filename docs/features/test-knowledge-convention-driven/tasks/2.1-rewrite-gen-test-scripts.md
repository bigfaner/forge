# 2.1: Rewrite gen-test-scripts skill

## Description
Rewrite the gen-test-scripts SKILL.md to use Convention files + Code Reconnaissance instead of Profile. Remove all `forge test detect`, `forge test get generate`, `forge test framework`, `forge test get template` calls. Add Convention loading, Code Reconnaissance extension, and compile gate with retry.

## Reference Files
- `plugins/forge/skills/gen-test-scripts/SKILL.md` — current skill (to rewrite)
- `docs/features/test-knowledge-convention-driven/design/tech-design.md` — I-6 gen-test-scripts interface
- `docs/features/test-knowledge-convention-driven/prd/prd-spec.md` — FS-1 through FS-4

## Acceptance Criteria
- SKILL.md contains zero references to `forge test detect`, `forge test get`, `forge test framework`
- Step 0: List `docs/conventions/` → load files matching `domains: [testing, ...]`
- Step 1: Code Reconnaissance — scan existing test files for framework patterns (imports, tags, naming)
- Step 2: Generate code using Convention + Reconnaissance + Skill methodology
- Step 3: Compile gate `just e2e-compile` → retry on failure (max 2)
- Convention validation: missing sections → warning + LLM defaults; malformed file → skip with warning
- No Convention found → output hint, proceed with LLM defaults
- Compile gate failure recovery: output error + file path + recovery suggestions

## User Stories
- Story 1 (non-default framework): Convention drives framework-specific code
- Story 3 (cold start): LLM defaults work without Convention
- Story 6 (compile gate recovery): actionable guidance on failure

## Hard Rules
- No `if framework == "go" then ...` branch logic in SKILL.md — framework differences are entirely described by Convention content
- Skill methodology (pipeline structure, output rules, tag lifecycle) stays in SKILL.md; framework knowledge moves to Convention

## Implementation Notes
- Skill prompt structure: Pipeline overview → Step 0 (Convention load) → Step 1 (Reconnaissance) → Step 2 (Generate) → Step 3 (Compile gate)
- Code Reconnaissance Fact Table extended with test framework columns: file-pattern, assertion-lib, build-tag, naming-pattern
- Convention validation rules per FS-1 in PRD
