# 3.1: Create /forge:test-guide skill

## Description
Create a new skill at `plugins/forge/skills/test-guide/` that guides users through generating Convention files. The skill detects project file signals, scans existing test files, extracts patterns, and writes a minimal Convention file after user confirmation.

## Reference Files
- `docs/features/test-knowledge-convention-driven/design/tech-design.md` — I-6 test-guide interface
- `docs/features/test-knowledge-convention-driven/prd/prd-spec.md` — FS-5
- `docs/features/test-knowledge-convention-driven/prd/prd-user-stories.md` — Story 2, Story 5

## Acceptance Criteria
- New skill created at `plugins/forge/skills/test-guide/SKILL.md`
- Step 1: Scan project for file signals (go.mod, package.json, Cargo.toml, etc.)
- Step 2: Scan existing test files → extract patterns (imports, tags, naming, assertion lib)
- Step 3: Present findings for user confirmation; cold start → present framework candidates
- Step 4: Write `docs/conventions/testing-<scope>.md` with minimal structure (Framework + Assertion + Tags + Result Format)
- Generated Convention file includes `<!-- auto-generated by forge:test-guide -->` marker
- Existing Convention file → show diff, ask user to confirm update or keep

## User Stories
- Story 2 (Convention bootstrap): guided detection with minimal effort
- Story 5 (multi-framework): separate Convention files per framework

## Hard Rules
- No code execution — skill is LLM-driven file analysis and generation
- Convention file must follow fixed section structure defined in tech-design.md Data Models

## Implementation Notes
- File signal detection patterns: go.mod → Go, package.json → JavaScript/TypeScript, Cargo.toml → Rust, pom.xml → Java, etc.
- Test file patterns: *_test.go, test_*.py, *.spec.ts, *Test.java, etc.
- Warm start analysis: extract import statements → identify assertion library, extract build tags → identify tag convention
- Cold start: list mainstream frameworks per detected language, ask user to select
