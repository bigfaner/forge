# 2.1: Add Execution Workflow to breakdown templates + remove noTest

## Description

Add `## Execution Workflow` section to all 10 breakdown-tasks templates. Each workflow specifies concrete steps appropriate to the task type (TDD, execution, doc-gen, etc.). Remove `noTest` from all frontmatter. The default template (`task.md`) contains the TDD + Quality Gate workflow that serves as the fallback for task-executor.

## Implementation Notes

Workflow content for each template type (from tech-design §Template Hierarchy):

**task.md** (DEFAULT — most critical, serves as fallback):
Must contain the full TDD + Quality Gate cycle currently hardcoded in task-executor Steps 2-3. This is what old tasks fall back to.

**run-e2e-tests.md**:
Must specify: run tests, classify failures (flaky/environment/defect), create fix tasks for defects, DO NOT re-run or enter TDD cycle. This is the template that motivated this entire feature (14-min waste).

**gen-test-cases.md**, **consolidate-specs.md**, **phase-summary-task.md**, **eval-test-cases.md**:
These were `noTest: true` because they don't need TDD. Their workflows should specify: generate/extract content, verify output exists, stop. No quality gate.

**gen-test-scripts.md**:
Was `noTest: false` but not a TDD task. Workflow: generate scripts, verify TypeScript compilation, stop.

Follow the workflow content model (W1-W5) from tech-design for each:
- W1: Has ## Execution Workflow heading
- W2: Content is non-empty
- W3: Has explicit stop condition
- W4: No open-ended instructions
- W5: Specifies success/failure criteria
