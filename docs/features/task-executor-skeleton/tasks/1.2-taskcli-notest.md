# 1.2: Remove NoTest from task-cli (Go)

## Description

Remove the `NoTest` field and all related logic from the task-cli Go codebase. This includes: struct field removal, claim output field removal, coverage auto-set logic removal, quality gate skip logic removal, error message updates, and test updates.

## Reference Files

- `task-cli/pkg/task/types.go` — Task and TaskState structs (remove NoTest field)
- `task-cli/internal/cmd/claim.go` — Claim output (remove NO_TEST print)
- `task-cli/internal/cmd/record.go` — Record logic (remove 3 noTest behaviors)
- `task-cli/internal/cmd/errors.go` — Error messages (update ErrNoTestEvidence)
- `task-cli/internal/cmd/record_test.go` — Unit tests (update/remove noTest cases)
- `task-cli/internal/cmd/integration_test.go` — Integration tests (update/remove noTest cases)
- `docs/features/task-executor-skeleton/design/tech-design.md` §Interface 3, 4 — Typed changes

## Acceptance Criteria

- [ ] `types.go`: `NoTest bool` removed from both Task struct (~line 32) and TaskState struct (~line 120)
- [ ] `claim.go`: `NoTest: t.NoTest` copy removed (~line 128), `PrintField("NO_TEST", ...)` removed (~line 321)
- [ ] `record.go`: coverage auto-set logic removed (~lines 113-116), quality gate skip removed (~lines 121-124), noTest formatting removed (~lines 380, 393-395)
- [ ] `errors.go`: ErrNoTestEvidence message updated — no longer suggests "add noTest:true to the task template"
- [ ] `go build ./...` passes
- [ ] `go test ./...` passes (all updated tests)
- [ ] `grep -r "noTest\|NO_TEST\|NoTest" task-cli/` → zero matches (excluding test fixture expectations that verify the field is gone)

## User Stories

**Story 3** (Forge Maintainer):
- AC1: grep noTest/NO_TEST zero match → this task removes all Go code references
- AC2: no noTest conditional branches in task-cli → all three behaviors removed (coverage auto-set, QG skip, formatting)

## Implementation Notes

**Types change** (types.go):
```go
// BEFORE:
type Task struct {
    // ... other fields
    NoTest bool `json:"noTest,omitempty"`
}
type TaskState struct {
    // ... other fields
    NoTest bool `json:"noTest,omitempty"`
}
// AFTER: remove NoTest from both structs
```

**Claim change** (claim.go):
```go
// BEFORE:
NoTest: t.NoTest,
// AFTER: remove this line

// BEFORE:
PrintField("NO_TEST", strconv.FormatBool(t.NoTest))
// AFTER: remove this line
```

**Record changes** (record.go — 3 sites):
1. Remove coverage auto-set block: `if t.NoTest && rd.Coverage >= 0 ...`
2. Remove QG skip: change `if rd.Status == "completed" && !recordForce && !t.NoTest` to `if rd.Status == "completed" && !recordForce`
3. Remove noTest formatting: `formatTestsExecuted` and `formatCoverage` no longer need noTest parameter

**Error change** (errors.go):
- ErrNoTestEvidence should suggest using `--force` instead of `noTest: true`

**Test updates**: Remove/update all test cases that test noTest-specific behavior. Tests that verify noTest=false behavior should still pass (they test the default path which is now the only path).
