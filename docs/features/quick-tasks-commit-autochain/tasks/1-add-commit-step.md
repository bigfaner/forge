# 1: Add Step 8 (Commit) to quick-tasks SKILL.md

## Description
quick-tasks skill generates planning artifacts (task .md files, index.json, manifest.md) but never commits them. run-tasks executor only commits code files it modifies, leaving planning docs untracked. Add a Step 8 to quick-tasks SKILL.md that commits all generated planning artifacts after Step 7 validation passes.

## Acceptance Criteria
- [ ] SKILL.md contains a new Step 8 titled "Commit" after Step 7 (Validate)
- [ ] Step 8 uses `git add` with explicit file paths (task .md files, index.json, manifest.md), NOT `git add -A`
- [ ] Step 8 only executes after Step 7 validation passes (no commit on validation failure)
- [ ] Commit message follows Conventional Commits format (e.g., `docs(features): add quick-tasks planning artifacts`)

## Hard Rules
- Must read `docs/conventions/forge-distribution.md` before modifying plugin files
- `git add` must target specific paths only — never `git add -A` or `git add .`

## Implementation Notes
- The commit path is the entire `docs/features/<slug>/` directory generated by Steps 3-7
- The commit should include: task .md files, index.json, manifest.md, and any auto-generated stage-gate files
- Consider dirty working tree scenario: only stage planning artifacts, leave other changes unstaged
