# 2.1: Create docs/decisions/ directory structure

## Description
创建 `docs/decisions/` 集中决策归档目录，包含 `manifest.md` 索引文件和 8 个类型模板文件，作为决策归档的数据存储基础设施。

## Reference Files
- `docs/features/feat-log-decisions/design/tech-design.md` §Data Models — Model 2/3 定义了文件结构
- `docs/features/feat-log-decisions/prd/prd-spec.md` §5.1 — 目录布局和字段说明

## Acceptance Criteria
- [ ] `docs/decisions/manifest.md` 存在，含 frontmatter（updated 字段）、Categories 表（8 行）、Recent Decisions 表（空）
- [ ] 以下 8 个类型文件存在，每个含标题行和空表格（Date/Feature/Decision/Rationale/Source 列）：
  - `architecture.md`
  - `interface.md`
  - `data-model.md`
  - `dependencies.md`
  - `error-handling.md`
  - `testing.md`
  - `security.md`
  - `local-dev-deployment.md`
- [ ] manifest.md Categories 表中 Decisions 列初始值为 0，LastUpdated 为 `-`

## Implementation Notes
- manifest.md 的 Categories 表需包含列：Category、File、Decisions、Last Updated
- manifest.md 的 Recent Decisions 表需包含列：Date、Feature、Category、Decision、Source
- 类型文件初始内容格式参考 tech-design.md §Model 3
