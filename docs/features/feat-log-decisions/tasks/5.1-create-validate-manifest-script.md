# 5.1: Create validate-manifest CI script

## Description
创建 `scripts/validate-manifest.sh` 脚本，解析 `docs/decisions/` 下所有类型文件的实际条目数，与 `manifest.md` 中 Decisions 列的计数比对，不一致则报错退出（exit 1）。

## Reference Files
- `docs/features/feat-log-decisions/design/tech-design.md` §Testing Strategy — validate-manifest CI 测试层
- `docs/features/feat-log-decisions/prd/prd-spec.md` §监控需求 — 脚本行为描述
- `docs/features/feat-log-decisions/design/tech-design.md` §Open Questions — 脚本路径待确认（scripts/ vs plugins/zcode/scripts/）

## Acceptance Criteria
- [ ] `scripts/validate-manifest.sh` 文件存在且可执行（chmod +x）
- [ ] 脚本遍历 8 个类型文件，统计每个文件中表格数据行数（排除表头行）
- [ ] 脚本读取 `docs/decisions/manifest.md` Categories 表中对应的 Decisions 计数
- [ ] 计数一致时脚本 exit 0，并输出 "manifest OK"
- [ ] 计数不一致时脚本 exit 1，并输出具体不一致的类型名和差值
- [ ] 在空目录（所有计数为 0）时脚本 exit 0

## Implementation Notes
- 脚本路径选择 `scripts/`（根目录下），与 Open Questions 中的默认选项一致
- 统计行数时需跳过表头行（`| Date | Feature |...`）和分隔行（`|------|...`）
- 可用 `grep -c "^|" file` 减去 2（表头+分隔行）得到数据行数
