# 4.1: Update hooks.json + agent + command file references

## Description

Update all `task <command>` references in hooks.json, the hooks guide, the task-executor agent file, and command files to use new `forge` command names.

## Reference Files

- `design/tech-design.md` §Appendix — Phase 4 Reference Update Map (hooks.json, hooks guide, agent files, command files tables)
- Current: `plugins/forge/hooks/hooks.json`, `plugins/forge/hooks/guide.md`, `plugins/forge/agents/task-executor.md`
- Current: `plugins/forge/commands/execute-task.md`, `plugins/forge/commands/run-tasks.md`, `plugins/forge/commands/git-checkout.md`, `plugins/forge/commands/quick.md`

## Acceptance Criteria

- [ ] `hooks.json`: `"task cleanup"` → `"forge cleanup"` (2 occurrences), `"task all-completed"` → `"forge quality-gate"`
- [ ] `hooks/guide.md`: `task record` → `forge task submit`, `task claim` → `forge task claim`, `task feature` → `forge feature`
- [ ] `agents/task-executor.md`: `task prompt` → `forge prompt get-by-task-id`, `task claim` → `forge task claim`, `task status` → `forge task status`
- [ ] `commands/execute-task.md`: all task command refs → forge equivalents, remove `task template` (deleted)
- [ ] `commands/run-tasks.md`: same as execute-task.md
- [ ] `commands/git-checkout.md`: `task feature` → `forge feature`
- [ ] `commands/quick.md`: `task validate` → `forge task validate-index`
- [ ] JSON files parse correctly after changes (`jq . <file>` succeeds)

## User Stories

- Story 4: hooks.json updated so SessionEnd/Stop/PreToolUse hooks call correct `forge` commands
- Story 3: Agent command files reference `forge task submit` for result submission

## Implementation Notes

Use the Phase 4 Reference Update Map from the design doc for exact old→new replacements. Each replacement follows the naming convention: task-group commands become `forge task <new-name>`, top-level commands become `forge <name>`. The `task template` reference in execute-task.md and run-tasks.md should be removed entirely since the command is deleted.
