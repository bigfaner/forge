# 1.1: Module/directory/binary rename + import path update

## Description

Rename the Go module, directory structure, and binary name from `task`/`task-cli` to `forge`/`forge-cli`. Update all Go import paths across the entire codebase. This is the foundational rename that all subsequent phases build upon.

## Reference Files

- `design/tech-design.md` §Architecture — File Structure Changes
- `design/tech-design.md` §Data Models — Version Name Change, Module Path Change
- Current: `task-cli/go.mod`, `task-cli/cmd/task/main.go`, `task-cli/pkg/version/version.go`

## Acceptance Criteria

- [ ] `task-cli/` directory renamed to `forge-cli/`
- [ ] `go.mod` module path changed from `task-cli` to `forge-cli`
- [ ] All Go import paths updated: `task-cli/...` → `forge-cli/...`
- [ ] `cmd/task/` renamed to `cmd/forge/` with `main.go` Use field set to `"forge"`
- [ ] `pkg/version/version.go` Name constant changed from `"task"` to `"forge"`
- [ ] `scripts/version.txt` updated to `3.0.0`
- [ ] `scripts/install-local.ps1` and `scripts/install-local.sh` updated with `forge` references
- [ ] `go build ./...` compiles without errors
- [ ] All existing tests pass with new import paths

## User Stories

- Story 1 (AI agent discovers commands): Binary rename from `task` to `forge` is the foundation for all command discovery.
- Story 3 (AI agent submits result): Module rename ensures correct binary is built and installed.

## Implementation Notes

This is a mechanical rename. Use IDE refactoring or `sed` to update all import paths atomically. Verify no `task-cli` references remain in Go source after the rename. The `go.mod` module path change propagates to every file with an import statement.
