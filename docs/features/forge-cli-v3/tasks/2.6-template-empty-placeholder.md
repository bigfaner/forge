# Task 2.6: Fix empty template variable rendering

## Summary

When `PHASE_SUMMARY` is empty (phase 1 tasks, or no previous phase summary exists), 8 prompt templates render `If `` is non-empty` as a nonsensical residual sentence, and leave an empty line where `{{PHASE_SUMMARY}}` stood alone.

## Problem

In `forge-cli/pkg/prompt/data/`, 8 templates share this pattern:

```
{{PHASE_SUMMARY}}
...
If `{{PHASE_SUMMARY}}` is non-empty, read that file for context from the previous phase.
```

`prompt.go:103` does `strings.ReplaceAll(result, "{{PHASE_SUMMARY}}", phaseSummaryLine)`. When `phaseSummaryLine` is empty:

1. Line 4 (`{{PHASE_SUMMARY}}` alone) → becomes an empty line
2. The conditional sentence → becomes `If `` is non-empty, read that file for context from the previous phase.`

The same issue applies to `{{SCOPE}}` in `just compile {{SCOPE}}` (trailing space), though that is cosmetic only.

## Affected Files

- `forge-cli/pkg/prompt/prompt.go` — replacement logic
- 8 template files in `forge-cli/pkg/prompt/data/`:
  - `implementation.md`
  - `doc-generation-summary.md`
  - `doc-generation-consolidate.md`
  - `test-pipeline-eval-cases.md`
  - `test-pipeline-gen-cases.md`
  - `test-pipeline-gen-scripts.md`
  - `test-pipeline-graduate.md`
  - `test-pipeline-run.md`
  - `test-pipeline-verify-regression.md`

## Acceptance Criteria

1. When a template variable is empty, the line containing only that variable is removed entirely
2. When a template variable is empty, any line containing a conditional sentence referencing that variable in backticks is removed entirely
3. When the variable has a value, behavior is unchanged (variable replaced, conditional sentence preserved with the value)
4. `just compile`, `just fmt`, `just lint`, `just test` all pass
5. Existing tests pass; new test(s) cover the empty-variable and non-empty-variable cases
