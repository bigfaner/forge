## Implementation Notes

Each e2e subcommand follows the same pattern: resolve profile → validate → dispatch to profile-specific handler. The `pkg/e2e/` action functions (implemented in task 3.1) handle the actual execution. The command files are thin wrappers that parse flags and call the action functions. Profile dispatch uses a simple `switch` statement — no interface/registry needed per design decision.
