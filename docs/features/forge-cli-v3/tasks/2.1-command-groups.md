# 2.1: Create command group parents + restructure root.go

## Description

Create 5 Cobra command group parents (task, e2e, forensic, profile, prompt), restructure root.go to register them, and move existing subcommands under their respective groups. Move cleanup, quality-gate (currently all-completed), verify-task-done (currently verify-completion), feature, and version to top-level. Split prompt.go into parent + subcommand. Move validate-specs into the e2e group.

- Story 2: `forge prompt get-by-task-id` structured under prompt group
- Story 4: cleanup, quality-gate, verify-task-done accessible as top-level commands for hooks
- Story 7: forensic commands grouped under `forge forensic`
- Story 8: profile commands grouped under `forge profile`

## Implementation Notes

The forensic and profile commands already have subcommand structure in the current codebase — they just need to be registered under group parents instead of directly in root.go. The prompt split is the most nuanced change: the current `prompt.go` becomes `prompt_parent.go` (empty parent) + `prompt_get.go` (the actual get-by-task-id logic). `validate_specs.go` move to e2e group is a file rename + re-registration, no logic change.
