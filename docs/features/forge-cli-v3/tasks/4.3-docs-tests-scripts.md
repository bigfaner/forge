## User Stories

- Story 1: Documentation shows correct `forge` command examples for AI agents and developers

## Implementation Notes

The documentation files contain the most references (OVERVIEW.md has ~15 unique command references). The ASCII diagram in WORKFLOW.md uses labels like `task claim`, `task record` — these need updating to `forge task claim`, `forge task submit`. Go test files that test CLI behavior need both the binary name (`"task"` → `"forge"`) and subcommand paths updated. Install scripts reference the binary name and module path.
