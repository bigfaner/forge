# 1.6: Extend task claim to output TYPE field

## Description

扩展 `task-cli/internal/cmd/claim.go`，在 claim 输出中新增 `TYPE` 字段（与现有 `MAIN_SESSION`、`NO_TEST` 同格式），并将 `Task.Type` 写入 `TaskState`。run-tasks 和 execute-task 需要从 claim 输出中读取 TYPE 来决定路由。

## Files Created/Modified

| Action | File |
|--------|------|
| Modify | `task-cli/internal/cmd/claim.go` |
| Modify | `task-cli/internal/cmd/claim_test.go` |

## Acceptance Criteria

- [ ] `task claim` 输出包含 `TYPE: <type-value>` 行（格式与 `MAIN_SESSION: true/false` 一致）
- [ ] `TaskState` 的 `Type` 字段在 claim 时从 `Task.Type` 复制
- [ ] type 为空时输出 `TYPE: ` 空值（不报错，向后兼容）
- [ ] 现有 claim 测试仍通过
- [ ] `go test ./internal/cmd/...` 通过

## User Stories

Story 6（execute-task 路由与 run-tasks 一致）：run-tasks 和 execute-task 均从 claim 输出读取 TYPE 字段来决定路由。

## Implementation Notes

- 参照 `printTaskDetails()` 中 `PrintField("MAIN_SESSION", ...)` 的模式新增 `PrintField("TYPE", t.Type)`
- `TaskState.Type = t.Type` 在 `executeClaim()` 的 state 构建处赋值
- 输出顺序建议：TYPE 紧跟在 MAIN_SESSION 之后
