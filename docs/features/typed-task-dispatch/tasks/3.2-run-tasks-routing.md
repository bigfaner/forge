# 3.2: Update run-tasks.md routing to use task prompt

## Description

更新 `plugins/forge/commands/run-tasks.md`，将 Step 2 dispatch 逻辑替换为调用 `task prompt <id>` 合成 prompt 后 dispatch 给 task-executor。移除 error-fixer dispatch，改为 `task prompt <id> --fix-record-missed`。新增 eval-cases 永久例外路由。

## Files Created/Modified

| Action | File |
|--------|------|
| Modify | `plugins/forge/commands/run-tasks.md` |

## Acceptance Criteria

- [ ] Step 2 调用 `task prompt <TASK_ID>` 获取 prompt
- [ ] `task prompt` exit 非零时：将 stderr 写入 blockedReason，`task status <KEY> blocked`，continue loop
- [ ] type == "test-pipeline.eval-cases" 时：在主会话中直接按 prompt 执行（不 dispatch subagent）
- [ ] 其他类型：`Agent(forge:task-executor, prompt=<stdout>)`，不再传 TASK_FILE、NO_TEST 参数
- [ ] record 缺失恢复：调用 `task prompt <TASK_ID> --fix-record-missed` → `Agent(forge:task-executor, prompt=<stdout>)`
- [ ] 移除所有对 `forge:error-fixer` 的 dispatch 调用
- [ ] claim 输出解析新增 TYPE 字段提取
- [ ] MAIN_SESSION 路由保留（eval-cases 通过 TYPE 判断，不再依赖 MAIN_SESSION 字段）

## User Stories

Story 6（execute-task 路由与 run-tasks 一致）：run-tasks 路由逻辑是 execute-task 的参照基准。
Story 7（error-fixer 废弃后等价覆盖）：run-tasks 不再引用 forge:error-fixer。

## Implementation Notes

新 Step 2 伪代码：
```
1. 从 claim 输出提取 TYPE
2. 运行: task prompt <TASK_ID>
   - 捕获 stdout 为 SYNTHESIZED_PROMPT
   - 捕获 stderr 为 PROMPT_ERROR
   - 检查 exit code
3. if exit != 0:
   - task status <KEY> blocked (with blockedReason=PROMPT_ERROR)
   - continue loop
4. if TYPE == "test-pipeline.eval-cases":
   - 在主会话中直接按 SYNTHESIZED_PROMPT 执行
5. else:
   - Agent(forge:task-executor, prompt=SYNTHESIZED_PROMPT)
```

record 缺失恢复（替换原 error-fixer dispatch）：
```
task prompt <TASK_ID> --fix-record-missed → Agent(forge:task-executor, prompt=<stdout>)
```
