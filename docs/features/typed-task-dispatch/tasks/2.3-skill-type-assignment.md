# 2.3: Add Type Assignment rules to breakdown-tasks and quick-tasks skills

## Description

在 `plugins/forge/skills/breakdown-tasks/SKILL.md` 和 `plugins/forge/skills/quick-tasks/SKILL.md` 的 Step 3（Create Task Files）中新增 Type Assignment 规则表，确保生成的任务自动包含正确的 type 字段。

## Files Created/Modified

| Action | File |
|--------|------|
| Modify | `plugins/forge/skills/breakdown-tasks/SKILL.md` |
| Modify | `plugins/forge/skills/quick-tasks/SKILL.md` |

## Acceptance Criteria

- [ ] breakdown-tasks SKILL.md Step 3 包含完整的 Type Assignment 规则表（11 种类型 + fallback）
- [ ] quick-tasks SKILL.md Step 3 包含相同的 Type Assignment 规则表
- [ ] 规则表包含 fallback：无法匹配时 type = "implementation"，stderr 输出警告
- [ ] 规则表与 `pkg/prompt.InferType()` 的推断规则完全一致（单一来源）

## User Stories

Story 5（breakdown-tasks 自动设置 type）：
- Given 一个完整的 tech-design 文档
- When 执行 breakdown-tasks
- Then 生成的 index.json 中所有任务均包含 type 字段，`task validate` 无报错，且 type 值与任务的实际类型一致

## Implementation Notes

Type Assignment 规则（与 InferType 保持一致）：

| 任务类型 | type 值 |
|---------|---------|
| 业务实现任务 | "implementation" |
| ID 以 .summary 结尾 | "doc-generation.summary" |
| ID 以 .gate 结尾 | "gate" |
| T-test-1 | "test-pipeline.gen-cases" |
| T-test-1b | "test-pipeline.eval-cases" |
| T-test-2 | "test-pipeline.gen-scripts" |
| T-test-3 | "test-pipeline.run" |
| T-test-4 | "test-pipeline.graduate" |
| T-test-4.5 | "test-pipeline.verify-regression" |
| T-test-5 | "doc-generation.consolidate" |
| ID 以 fix- 或 disc- 开头 | "fix" |
| 其余 | "implementation"（fallback，输出 warning） |

- 在 breakdown-tasks 的 Step 3 "Create Task Files" 小节末尾插入规则表
- 在 quick-tasks 的 Step 3 同位置插入相同规则表
