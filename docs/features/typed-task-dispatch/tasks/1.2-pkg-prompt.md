## Acceptance Criteria

- [ ] `Synthesize(SynthesizeOpts)` 对 11 种 type 各返回非空 prompt，无 `{{` 残留占位符
- [ ] `Synthesize` 对 `FixRecordMissed=true` 使用 `fix-record-missed.md` 模板
- [ ] `Synthesize` 对 `task.Type == ""` 返回 error
- [ ] `Synthesize` 对未注册 type 返回 error
- [ ] `PhaseDetect` 在 currentPhase > maxCompletedPhase AND currentPhase > 1 时返回 summary 路径
- [ ] `PhaseDetect` 在 summary 文件不存在时返回空字符串（非 error）
- [ ] `InferType` 对所有已知 ID 模式返回正确 type（table-driven 测试覆盖全部规则）
- [ ] `go test ./pkg/prompt/...` 通过，覆盖率 ≥ 80%

## User Stories

Story 1：fix 模板包含诊断→定位→修复→验证→提交五步流程，不含 TDD 步骤。
Story 2：新增类型只需在 `data/` 目录添加模板文件并在 `typeToTemplate` map 注册。
Story 3：`task prompt <id>` 的 stdout 来自此包的 `Synthesize()` 输出。

## Implementation Notes

- 模板内容参照现有 `plugins/forge/agents/task-executor.md` 和 `plugins/forge/agents/error-fixer.md` 逐条提取
- `fix.md` 模板必须承接 error-fixer 的全部能力：编译错误、测试失败、lint 问题
- `fix-record-missed.md` 模板专用于 record 缺失恢复，不入 index
- 占位符格式 `{{PLACEHOLDER}}` 与现有 `pkg/template/data/fix-task.md` 保持一致
- `PhaseDetect` 算法：扫描 index.json 中所有 completed 业务任务（排除 .gate/.summary/T- 前缀），取最大 phase 编号
- 依赖方向：`pkg/prompt` → `pkg/task`、`pkg/feature`、`pkg/project`（不可反向）
