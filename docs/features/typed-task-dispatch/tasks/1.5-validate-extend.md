# 1.5: Extend task validate with type field validation

## Description

扩展 `task-cli/internal/cmd/validate.go` 中的 `validateTasks()` 方法，新增 type 字段合法性验证和 mainSession 一致性检查。

## Files Created/Modified

| Action | File |
|--------|------|
| Modify | `task-cli/internal/cmd/validate.go` |
| Modify | `task-cli/internal/cmd/validate_test.go` |

## Acceptance Criteria

- [ ] type 字段缺失时：validate 报 error（不是 warning）
- [ ] type 为非法值时：validate 报 error，错误信息含具体非法值
- [ ] mainSession=true 且 type 不是 "test-pipeline.eval-cases" 时：validate 报 warning
- [ ] 现有所有 validate 测试仍通过（无回归）
- [ ] `go test ./internal/cmd/...` 通过

## User Stories

Story 4（迁移旧 index.json）：
- Then `task validate` 对迁移后的 index.json 无报错

Story 5（breakdown-tasks 自动设置 type）：
- Then `task validate` 对生成结果无报错

## Implementation Notes

- 在 `validateTasks()` 中新增两个检查块，参照现有 `validStatus`、`validPriority` 的模式
- 使用 `task.ValidTypes` map（在 1.1 任务中定义）进行合法性检查
- type 缺失是 error（不是 warning），因为 type 是必填字段
- 测试用例：缺失 type、非法 type、mainSession+非eval-cases 三种场景
