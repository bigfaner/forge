# 4.1: Deprecate error-fixer agent and remove orphan references

## Description

废弃 `plugins/forge/agents/error-fixer.md`，并确认 run-tasks.md、execute-task.md、guide.md 等文件中无孤立引用。error-fixer 的全部能力已在 Phase 3 迁移到 `type: fix` prompt 模板和 `--fix-record-missed` 模式。

## Files Created/Modified

| Action | File |
|--------|------|
| Modify | `plugins/forge/agents/error-fixer.md` |
| Modify | `docs/ARCHITECTURE.md` |

## Acceptance Criteria

- [ ] `error-fixer.md` 顶部添加 deprecated 标注，说明替代路径
- [ ] `grep -r "forge:error-fixer" plugins/forge/` 无结果（run-tasks、execute-task 均不引用）
- [ ] `grep -r "error-fixer" plugins/forge/commands/` 无结果
- [ ] `docs/ARCHITECTURE.md` 中 error-fixer 相关描述已更新（标注废弃，指向新路径）
- [ ] 无其他文件孤立引用 error-fixer

## User Stories

Story 7（error-fixer 废弃后等价覆盖）：
- Given index.json 中存在一个 type 为 "fix" 的任务
- When run-tasks 执行该任务
- Then task-executor 收到的 prompt 来自 `task prompt <id>`，包含诊断→定位→修复→验证→提交五步流程；run-tasks.md 中不存在对 forge:error-fixer 的 dispatch 调用

## Implementation Notes

- 不删除 error-fixer.md 文件（保留历史记录），在文件顶部添加 deprecated 标注
- deprecated 标注格式：`> **DEPRECATED**: This agent is no longer dispatched. Use \`type: fix\` tasks (routed via \`task prompt <id>\`) for compilation/test/lint fixes, and \`task prompt <id> --fix-record-missed\` for record recovery.`
- 用 grep 全面扫描所有引用，确保无遗漏
