# 1.1: Add Type/BlockedReason fields and type constants to pkg/task

## Description

在 `pkg/task/types.go` 中新增 `Type`、`BlockedReason` 字段到 `Task` 和 `TaskState` struct，并新增 11 种类型的常量和 `ValidTypes` map。这是 Phase 1 所有后续任务的基础。

## Files Created/Modified

| Action | File |
|--------|------|
| Modify | `task-cli/pkg/task/types.go` |
| Modify | `task-cli/pkg/task/types_test.go` |

## Acceptance Criteria

- [ ] `Task` struct 新增 `Type string \`json:"type,omitempty"\`` 字段
- [ ] `Task` struct 新增 `BlockedReason string \`json:"blockedReason,omitempty"\`` 字段
- [ ] `TaskState` struct 新增 `Type string \`json:"type,omitempty"\`` 字段
- [ ] 11 个 type 常量已定义（TypeImplementation、TypeDocGenerationSummary 等）
- [ ] `ValidTypes map[string]bool` 包含全部 11 个合法值
- [ ] 现有 JSON 序列化/反序列化测试仍通过（omitempty 不破坏现有 index.json）
- [ ] `go test ./pkg/task/...` 通过，覆盖率 ≥ 80%

## User Stories

Story 1（非编码任务获得正确流程）：type 字段是路由决策的基础。
Story 2（新增类型只需一个模板文件）：ValidTypes map 是注册新类型的入口。

## Implementation Notes

- `omitempty` 是必须的：允许现有不含 type 字段的 index.json 正常加载，不破坏向后兼容
- `BlockedReason` 由 run-tasks 写入，不由 task-cli 内部逻辑写入
- 参考现有 `MainSession`、`NoTest` 字段的模式
