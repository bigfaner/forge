# 1.4: Implement task migrate command

## Description

新建 `task-cli/internal/cmd/migrate.go`，实现 `task migrate` 命令。命令读取当前 feature 的 index.json，对每个任务调用 `pkg/prompt.InferType()` 填充 type 字段，保留所有现有任务状态。

## Acceptance Criteria

- [ ] `task migrate` 对所有已知 ID 模式推断出正确 type，`task validate` 通过
- [ ] `task migrate` 在存在 in_progress 任务时：exit 1，stderr 含提示，index.json 不修改
- [ ] `task migrate` 完成后 stdout 输出 "Migrated N tasks. Run task validate to verify."
- [ ] 迁移后所有任务的 status 保持不变
- [ ] `rootCmd.AddCommand(migrateCmd)` 已注册
- [ ] `go test ./internal/cmd/...` 通过

## User Stories

Story 4（迁移旧 index.json）：
- Given 一个不含 type 字段的旧 index.json，且所有任务均为非 in_progress 状态
- When 执行 `task migrate`
- Then 所有任务均填充了正确的 type 字段，任务状态保持不变，`task validate` 无报错

## Implementation Notes

- 参照 `internal/cmd/add.go` 的命令结构模式
- 迁移前检查：任何 in_progress 任务 → 报错退出，不修改文件
- `InferType` 在 `pkg/prompt` 包中实现（1.2 任务），此处直接调用
- 迁移是幂等的：对已有 type 字段的任务，覆盖写入（不跳过）
