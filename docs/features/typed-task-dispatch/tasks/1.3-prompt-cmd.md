# 1.3: Implement task prompt command

## Description

新建 `task-cli/internal/cmd/prompt.go`，实现 `task prompt <id> [--fix-record-missed]` 命令。命令从 index.json 读取任务，调用 `pkg/prompt.Synthesize()`，将结果输出到 stdout。

## Acceptance Criteria

- [ ] `task prompt <id>` stdout 为非空合成 prompt，stderr 为空，exit 0
- [ ] `task prompt <id>` 在 type 字段缺失时：exit 1，stdout 为空，stderr 含 "type field missing"
- [ ] `task prompt <id>` 在 type 未注册时：exit 1，stdout 为空，stderr 含 "unknown type"
- [ ] `task prompt <id> --fix-record-missed` 使用 fix-record-missed 模板
- [ ] 命令在 500ms 内完成（本地文件读取 + 字符串替换，无网络调用）
- [ ] `rootCmd.AddCommand(promptCmd)` 已注册
- [ ] `go test ./internal/cmd/...` 通过

## User Stories

Story 3（独立检查合成结果）：
- Given 当前 feature 有一个 in_progress 任务（任意 type）
- When 执行 `task prompt <id>`
- Then stdout 输出完整合成 prompt，包含任务 ID、scope、执行步骤；命令在 500ms 内完成

## Implementation Notes

- 参照 `internal/cmd/template.go` 的命令结构模式
- 错误输出到 stderr，成功输出到 stdout（不混用）
- 命令签名：`task prompt <id> [--fix-record-missed]`
- 通过 `RequireFeature()` 获取 feature slug，通过 `LoadIndex()` 读取任务
