# 1.1: Add e2e-setup and e2e-verify targets to init-justfile

## Description

Extend `plugins/forge/commands/init-justfile.md` with two new just targets:
- `e2e-setup`: idempotent install of e2e deps (npm install + playwright install chromium)
- `e2e-verify --feature <slug>`: checks for unresolved `// VERIFY:` markers, exit 1 if found

Both recipes must be added to:
1. The Standard Target Contract table
2. The recipe template section (verbatim, ready to copy into a justfile)
3. The Step 4 Output Confirmation example

## Reference Files

- `plugins/forge/commands/init-justfile.md` — file to modify
- `docs/features/justfile-e2e-integration/design/tech-design.md` — Interface 1 and Interface 2 sections contain verbatim recipe templates

## Acceptance Criteria

- [ ] Standard Target Contract table contains `e2e-setup` and `e2e-verify` rows
- [ ] `e2e-setup` recipe template present with `set -euo pipefail`, package.json check, idempotent node_modules check, playwright install, and success output
- [ ] `e2e-verify` recipe template present with `[arg("feature", long)]`, empty-string guard, directory-existence check, grep scan, and correct exit codes
- [ ] Step 4 Output Confirmation lists both new targets
- [ ] `grep -c 'e2e-setup\|e2e-verify' plugins/forge/commands/init-justfile.md` >= 4

## User Stories

Story 1 / AC-1: Given init-justfile has been run, When Skill maintainer views the Standard Target Contract, Then e2e-setup and e2e-verify appear as named targets with descriptions.

## Implementation Notes

Use the verbatim recipe blocks from `design/tech-design.md` Interface 1 and Interface 2. Do not paraphrase — copy the exact recipe syntax including `set -euo pipefail`, the `[arg("feature", long)]` annotation (requires just >= 1.50.0), and all exit code paths.
